# テーマ10 共通仕様 ― マッチ3パズル（ロジック主体・軽描画 × 大量トゥイーン）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- ぷよ/ツムツム/Bejeweled 風のマッチ3パズル。盤面の宝石（gem）を隣と入れ替え、3つ以上揃うと消え、上から落ちて補充され連鎖する。
- **目的**: 性能比較。**描画は軽いがロジック（マッチ判定・連鎖・盤面状態管理）と大量の同時トゥイーン（落下/消滅アニメ）が主役**な、
  既存テーマ（描画/物理が重い）とは**対照的な負荷**を比較する。
- 性能比較の核は「**盤面サイズ（セル数）＝同時に動くトゥイーン数とマッチ走査コスト**」。
- **オートプレイ前提**: 決定的なソルバが毎ターン有効な手を自動で打つため、**マウス無しでベンチが回り続ける**。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。盤面は中央に正方形領域で配置。
- 盤面: **N x N グリッド**（N = 負荷値。初期 **12**）。セル1辺 = `floor(min(520 / N, 56))` px で動的に決定（盤面が常に中央に収まるように）。
  - 宝石は **6 種類**（色/形）。盤面・初手は固定シードで決定的に生成（生成直後に既存マッチが無い状態にする）。
- 時間はデルタタイム基準。トゥイーンも実時間（秒）で進める。

## ゲームの流れ（状態機械）
1. **IDLE**: 入力/オートソルバ待ち。
2. **SWAP**: 選んだ2セルを入れ替えるトゥイーン（~0.15s）。入れ替え後にマッチが無ければ**元に戻す**（無効手）。
3. **CLEAR**: マッチしたセルを消滅トゥイーン（縮小+フェード, ~0.2s）。**Score += 消えた数 × 10 × 連鎖係数**。
4. **FALL**: 上の宝石が空きへ落下するトゥイーン（~0.2s, 落下距離に比例）。上端から新規宝石を**決定的に補充**。
5. 落下後に新たなマッチがあれば **CLEAR へ戻り連鎖**（chain++）。無ければ IDLE。
- マッチ判定: 横/縦に**同種3つ以上連続**を全消し。判定は毎回盤面全走査（N×N）。

## 操作
- **クリック2回**（隣接2セルを選択）で手動スワップ（隣接のみ有効）。
- `Space`: **オートプレイ**トグル（初期 **ON**）。ON の間、**0.25s 間隔で決定的に「マッチを生む隣接スワップ」を探索して自動実行**。
  - 探索は決定的（盤面を左上から走査し、最初に見つかった有効手を打つ）。有効手が無ければ盤面を決定的にシャッフル。
- `+` / `-`: **盤面サイズ N を増減**。性能比較の主軸（N×N の走査と同時トゥイーン数が増える）。
- `R`: 盤面リセット（決定的に再生成）。

## ルール / 数値（全エンジン共通）
- 宝石種別 **6**。`Score` は消去数×10×連鎖係数（連鎖係数 = 連鎖段数, 1段目=1, 2段目=2 …）。
- トゥイーン時間: スワップ 0.15s / 消滅 0.2s / 落下は距離比例（基準 0.2s/セル, 加速付き or 線形どちらでも可）。
  - **同時に進行するトゥイーン数**が負荷の核。落下時は1列まとめてではなく**セル単位**でトゥイーンを張る（数を稼ぐ）。
- 補充宝石は固定シードのPRNGで決定的に生成（Math.random 不使用）。盤面初期化時は**初期マッチ無し**になるまで決定的に振り直す。
- これはパズルのため勝敗は無い（無限プレイ）。盤面が詰んだ（手が無い）ら決定的シャッフル。
- 各エンジンの**トゥイーン/アニメ機構の比較**が裏テーマ（Phaser Tweens / Pixi 手動 or ticker / Babylon Animation / LittleJS 手動補間 / Godot Tween）。
  - どのトゥイーン機構を使ったか、N を上げたときの破綻点を README に記載する。

## 盤面サイズ（負荷）
- N 初期 **12**、`+`/`-` で **±2**（下限 **6**・上限 **40**）。
  - N=40 で 1600 セル。落下/消滅が同時多発すると**同時トゥイーン数が数百〜千**に達する。
  - 走査コスト（マッチ判定 O(N²)）とトゥイーン更新/描画の両方が負荷。描画自体は単純な矩形/丸なので、
    **ロジックとトゥイーン管理のオーバーヘッド**がエンジン差として出る。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Board`（N x N = セル数）/ `Active tweens`（現在進行中のトゥイーン数）
- `State`（IDLE / SWAP / CLEAR / FALL）/ `Chain`（現在の連鎖段数）
- `Score` / `Moves`（実行した手数）/ `Auto`（ON/OFF）
- 画面下に操作ヒント行（クリック2回=スワップ / Space=オート / +/-=盤面 / R=リセット）。
