# テーマ9 共通仕様 ― アイソメトリック都市/農場（深度ソート × タイル奥行き描画）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 斜め見下ろし（アイソメトリック）のマップ上で、地面タイル・建物・木・歩き回るユニットを正しい前後関係で描く都市/農場風シーン。
- **目的**: 性能比較。**アイソメトリック投影での深度ソート（z-order / 奥のものを先に・手前を後に）＋ 可視カリング ＋ 多数の移動ユニット**を測る。
  - テーマ3のトップダウン直交カリングとは別軸の、**斜め投影＋毎フレームの前後ソート**コストを比較する。
- 性能比較の核は「**深度ソート対象スプライト数（タイル＋オブジェクト＋ユニット）**」。

## アイソメトリック投影（全エンジン共通の式）
- グリッド座標 `(gx, gy)`（タイル整数 or 連続値）→ 画面座標へ:
  - `screenX = (gx - gy) * (TILE_W / 2)`
  - `screenY = (gx + gy) * (TILE_H / 2)`
  - タイル菱形サイズ **TILE_W = 64, TILE_H = 32**。
- **深度キー** = `gx + gy`（同値はオブジェクト種別/設置順で安定ソート）。値が小さいほど奥（先に描く）、大きいほど手前（後に描く）。
  各エンジンの z-order 機構（Pixi `zIndex`+`sortableChildren` / Phaser `depth` / Babylon スプライト order / LittleJS 描画順 / Godot `YSort`）で実現する。
- スクリーン中央にワールド中心が来るよう、描画時に画面オフセット（カメラ）を加算する。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。
- マップ: **64 x 64 タイル**（決定的に生成・固定シード）。地面種別と静的オブジェクトを配列で保持。
  - 地面: `0=草(grass)`, `1=土/畑(soil)`, `2=水(water)`。
  - 静的オブジェクト（タイル上に乗る）: 木 / 家（建物）を決定的に散布。建物・木は地面より背が高く、深度ソートで前後が入れ替わる。
- **カメラ**: 矢印 `←→↑↓` / `WASD` でワールドをスクロール（画面オフセットを移動）。中心はワールド内にクランプ。
- **可視範囲のタイル/オブジェクトのみ描画（カリング）**すること。画面に入るアイソメ範囲だけを対象に深度ソートする。

## ユニット（負荷の主役）
- マップ上を**決定的にうろつく**ユニット（村人/動物）。半径 ~10、移動 **40 px/s 相当**（グリッド連続座標で移動）。
  - 各ユニットは決定的な目的地へ向かい、着いたら次の決定的な目的地へ（固定シードの巡回）。水タイルは避ける。
  - **ユニットは連続座標を持ち深度キー `gx+gy` が毎フレーム変わるため、毎フレーム再ソートが要る**（ここが比較の核）。
- ユニットと建物/木が前後に重なったとき、深度ソートが正しければ自然な遮蔽になる。

## 操作
- 移動: 矢印 / `WASD` でカメラスクロール。
- `+` / `-`: **歩き回るユニット数（負荷）を増減**。性能比較の主軸（再ソート対象が増える）。
- `G`: グリッド線表示トグル（任意・デバッグ可視化）。
- `R`: リセット。

## ルール / 数値（全エンジン共通）
- これは探索/箱庭デモのため勝敗は無い（代わりに描画統計をHUDに出す）。
- 深度ソートは**可視オブジェクトのみ**を対象にすること（全6万タイルをソートしない。カリング後に `gx+gy` 昇順で描く）。
- 地面タイルもアイソメ描画だが、地面同士は高さ0で重ならないため、**地面 → 高さのあるオブジェクト/ユニット**の順で
  描けば足りる（地面はソート不要、オブジェクト＋ユニットのみ `gx+gy` でソート）。実装はこの2層方式を推奨。
- 決定的生成（固定シード）で Math.random 不使用。

## ユニット数（負荷）
- 初期 **60** 体、`+`/`-` で **±20**（下限 **0**・上限 **2000**）。
  - 増やすほど「毎フレーム深度ソートし直す可視オブジェクト数」が増える。ソートアルゴリズム/実装の差が `FPS` に出る。

## アセット（`../assets/` から読み込み。アイソメ用に**菱形の足元**を持つ画像が望ましい）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `tile_grass.png` | 草地（アイソメ菱形 64x32） | 64x32 |
| `tile_soil.png` | 畑/土（菱形） | 64x32 |
| `tile_water.png` | 水（菱形） | 64x32 |
| `tree.png` | 木（背高・足元が菱形中心） | 48x64 |
| `house.png` | 家/建物（背高） | 64x64 |
| `villager.png` | 歩くユニット | 96x128（24x32、4列x4行: 下/上/左/右） |

- **画像が無い場合は図形フォールバック**（地面=菱形ポリゴン: 草=緑/土=茶/水=青 / 木=茶幹+緑円 / 家=灰菱形柱 / ユニット=橙丸）。
  - 図形でもアイソメ菱形（4頂点ポリゴン）で描き、奥行きが分かるようにする。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Tiles drawn`（可視地面タイル数）/ `Objects sorted`（深度ソートした可視オブジェクト＋ユニット数）
- `Units`（ユニット数 現在 / 設定値）
- カメラ中心のワールド座標 `(gx, gy)`
- 画面下に操作ヒント行（矢印/WASD=スクロール / +/-=ユニット数 / G=グリッド / R=リセット）。
