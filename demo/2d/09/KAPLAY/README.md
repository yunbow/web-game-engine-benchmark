# アイソメトリック都市/農場 — KAPLAY 版

テーマ9共通仕様（アイソメトリック都市/農場・深度ソート性能比較用）の **KAPLAY** 実装。
画像アセットが無くても菱形ポリゴン等の図形フォールバックで必ず起動します。

使用バージョン: **KAPLAY 3001.0.19**（CDN: `unpkg.com/kaplay@3001.0.19/dist/kaplay.js`）。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`9/`)をルートに HTTP サーバを立てて開く。

```bash
cd 9
python -m http.server 8000
# → http://localhost:8000/KAPLAY/
```
`file://` 直開きは画像が CORS で読めない（図形フォールバックでは起動する）。

## 操作
- スクロール: 矢印キー / WASD（カメラをワールド内でクランプ）
- `+` / `-`（テンキー +/- も可）: 歩き回るユニット数を ±20（初期60・下限0・上限2000）
- `G`: グリッド線表示トグル
- `R`: リセット（マップ再生成・ユニット数を初期化）

## 画面・HUD
- キャンバス: **960 x 540**（固定）。
- HUD（HTML オーバーレイ）: `FPS` / `Tiles drawn`（可視地面タイル数）/ `Objects sorted`（深度ソートした可視オブジェクト＋ユニット数）/ `Units`（現在 / 設定値）/ カメラ中心ワールド座標 `(gx, gy)` / 操作ヒント。

## ★ 深度ソートの実現方法（KAPLAY = ペインターズアルゴリズム）
KAPLAY には `z()` コンポーネントによる重ね順制御もありますが、毎フレーム数百〜数千スプライトの `z` を再設定するとシーングラフ走査コストが嵩みます。
本実装は PixiJS リファレンスの「手で並べた描画順」を最も忠実に映す **イミディエイトモード（手動描画順制御）** を採用しています:

- `k.onDraw()` の中で、可視範囲のオブジェクト/ユニットを配列 `drawList` に集める。
- その配列を **`depth = gx + gy` 昇順でソート**する（`drawList.sort(...)`）。
- ソート順に `drawSprite` / `drawPolygon`（フォールバック）を呼ぶ。**後に描いたものが手前に来る** KAPLAY の描画規則がそのまま「奥→手前」の深度ソートになる。
- 地面は高さ0で重ならないため **ソートせず**、`gx+gy` 昇順に走査して描くだけ（SPEC 推奨の 2 層方式）。これにより毎フレームのソート対象は「可視オブジェクト＋ユニット」だけに限定される。

## 実装メモ
- `kaplay({ canvas, width:960, height:540, global:false })` で初期化。名前空間 `k.*` を明示利用。
- KAPLAY の座標系は **Y 下向き・原点左上 = 画面座標と一致**するため座標変換が素直（three/A-Frame の Y 反転が不要）。カメラは `pushTranslate(offX, offY)` のワールド平行移動で表現。
- アイソメ投影は SPEC の式: `screenX=(gx-gy)*32`, `screenY=(gx+gy)*16`。深度キー `gx+gy`。
- マップは `mulberry32` 固定シード（`20250609` 地面 / `424242` オブジェクト）で決定的生成。`Math.random` 不使用。リファレンス PixiJS と同一手順。
- ユニットは各自固有シードの RNG で決定的に徘徊（水・場外を避け 40px/s 相当）。連続座標のため `gx+gy` が毎フレーム変化し、毎フレーム再ソートが要る（比較の核）。
- カリング: 画面四隅をグリッド座標へ逆投影し、外接矩形のタイルのみ描画。
- アセットは `loadSprite` を個別 try/catch、失敗したものだけ図形（`drawPolygon`/`drawCircle`/`drawRect`）にフォールバック。

## Codex / AI コーディング所感
- 描画・ループ・入力をミニマル API で一発生成しやすい。座標系が画面座標と一致するため、アイソメ投影をそのまま `drawSprite` に流せる。
- 罠: ① 深度ソートを `z()` コンポーネントで素朴に実装すると大量スプライトで失速しやすく、**イミディエイト + 手動ソート**の方針を明示しないと比較条件がブレる。② キー名が独自（`"left"`, `"kpadd"`, `"minus"`）。③ `pushTransform`/`popTransform` の対応漏れに注意。
