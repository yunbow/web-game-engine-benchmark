# テーマ12 共通仕様 ― フォーリングサンド / セルオートマトン（動的テクスチャ書き換え）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 砂・水・壁などのセルが落下・堆積・流動する「フォーリングサンド」型のセルオートマトン。マウスで素材を描き込み、上部のエミッタからも自動で降り注ぐ。
- **目的**: 性能比較。**格子セルのシミュレーション更新 ＋ 毎フレームの全面テクスチャ書き換え（CPUピクセルバッファ → GPUアップロード）**のスループットを測る。
  - テーマ1〜11は「スプライト/図形/ライト」を描くだけで、**毎フレーム動的にテクスチャ（ピクセル）を生成してアップロード**する処理は誰も行っていない。本テーマはその未計測軸を埋める。
- 性能比較の核は「**グリッド解像度（セル数）＝ 1フレームあたりのセル更新数 ＋ テクスチャ転送サイズ**」。

## テクスチャ書き換え方式（エンジンごとに“素直な選択”を使う ＝ これ自体が比較対象）
- 各フレーム、`COLS × ROWS` の **RGBA ピクセルバッファ**（`Uint8ClampedArray`/`ImageData`/`PackedByteArray` 等）に全セルの色を書き、**1枚のテクスチャへアップロード**して画面いっぱい（960x540）に**ニアレストネイバーで拡大描画**する。
  - **Phaser4**: `this.textures.createCanvas`（CanvasTexture）に `putImageData` → `refresh()`、または ImageData を貼ったテクスチャを毎フレーム更新。
  - **PixiJS**: `<canvas>`/ImageData をソースにした `Texture` を毎フレーム `source.update()`。Sprite を拡大表示。
  - **Babylon.js**: `RawTexture`（`update`）または `DynamicTexture`（`putImageData`）を全画面の正射影 Plane に提示。
  - **LittleJS**: オフスクリーン `<canvas>` に `putImageData` → `mainContext`/`overlayContext` で拡大 blit（または tile テクスチャ更新）。
  - **Godot**: `Image`（`PackedByteArray` から `set_data`/`create_from_data`）→ `ImageTexture.update(image)` を毎フレーム。`TextureRect`/`_draw` で拡大。
- いずれも**ピクセル補間 = ニアレスト**（ドットがくっきり見えるように）。使った機構を各 README に必ず記載する。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。
- グリッド: **COLS 列 × ROWS 行**（`ROWS = round(COLS * 540 / 960)`）。テクスチャは `COLS×ROWS`、表示は 960x540 へ拡大。
- セル素材: `0=空気(empty)`, `1=砂(sand)`, `2=水(water)`, `3=壁(wall, 不動)`。（任意で `4=石/静止物` を足してよい）
- 時間はデルタタイム基準だが、**シミュレーションは固定タイムステップ**（毎フレーム1ステップ更新）でよい。

## シミュレーション規則（全エンジン共通・決定的）
- **走査順**は決定的（例: 下の行から上へ。各行は左右交互スキャンにして偏りを抑える）。`Math.random` 不使用。乱択が要る箇所は決定的PRNGで。
- **砂(sand)**: 真下が空/水なら落下。塞がれていれば左下・右下（決定的に選択）へ。水とは入れ替わる（砂が沈む）。
- **水(water)**: 真下が空なら落下。塞がれていれば左下・右下、それも塞がれていれば左右へ広がる（決定的順）。
- **壁(wall)**: 不動。
- 場外（グリッド外）は壁扱い（落下が下端で止まる）。

## エミッタ（無入力でもベンチが回る）
- 上部に**決定的なエミッタを数個**配置し、砂/水を毎フレーム少量供給する（マウス無しでも常にセルが動く＝ベンチ安定）。位置・素材は固定シードで決定的。

## 操作
- **左ドラッグ**: 現在のブラシ素材を描き込む（ブラシ半径数セル）。
- **右ドラッグ**: 消去（empty で塗る）。
- `1`/`2`/`3`（/`4`）: ブラシ素材を 砂 / 水 / 壁（/ 石）に切替。
- `+` / `-`: **グリッド解像度（COLS）を増減**。性能比較の主軸（セル数＝更新数＋転送量）。
- `C`: 全消去（エミッタは残す）。 `R`: リセット（決定的初期状態へ）。

## グリッド解像度（負荷）
- `COLS` 初期 **160**（→ ROWS=90, 14400 セル, セル6px相当）、`+`/`-` で **±40**（下限 **80**・上限 **640**）。
  - 上限 640（→ ROWS=360, 230400 セル, セル1.5px相当）まで上げると、**セル更新コスト**と**毎フレームのテクスチャ転送（COLS×ROWS×4バイト）**の双方が効く。
  - グリッドを変えたら状態は決定的に作り直す（エミッタ位置は比率で再配置）。

## アセット（`../assets/` から読み込み）
- **本テーマは原則アセット不要**（セルの色はコードで定義し、毎フレーム生成するため）。素材色は以下を基準にフォールバック/標準とする。
  - 砂 = 砂色（例 `#d9c067`系のランダム濃淡）, 水 = 半透明青（例 `#3a7bd5`系）, 壁 = 灰（`#888`）, 空気 = 背景暗色（`#0b0d12`）, 石 = 濃灰。
- 任意で背景テクスチャ等を置いてもよいが、必須アセットは無し（`assets/.gitkeep` のみ）。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Grid`（`COLS x ROWS` = セル数）
- `Active`（空気以外のセル数、または当該フレームで移動したセル数）
- `Brush`（現在のブラシ素材: sand/water/wall/…）
- `Upload`（使ったテクスチャ更新機構: 例 "ImageData→CanvasTexture" / "RawTexture.update" / "ImageTexture.update"）
- 画面下に操作ヒント行（左/右ドラッグ=描画/消去 / 1-3=素材 / +/-=解像度 / C=クリア / R=リセット）。
