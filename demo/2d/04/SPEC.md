# テーマ4 共通仕様 ― ブロック崩し（マルチボール）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 古典的なブロック崩し（Breakout / Arkanoid 風）。下部のパドルを左右に動かし、ボールを反射させて上部のブロックを破壊する。
- **目的**: 性能比較。**多数のボール ×（最大数百個）のブロックの反射・当たり判定**を毎フレーム回し、衝突処理スループットの限界を測る。

## 画面・基本設定
- キャンバス: **960 x 540**（16:9, 固定）。背景は暗色 or `bg_breakout.png`。
- 60FPS目標。デルタタイム基準で更新。
- 座標は画面座標（左上原点, y下向き）。※LittleJS は y 上向きなので内部で反転すること（他テーマと同様）。

## 操作
- パドル移動: 矢印 `←`/`→` または `A`/`D`（水平のみ、画面内クランプ）。マウス左右追従も任意で可。
- ボール: **自動発射**（手動ロックなし。負荷比較のため常にプレイ継続）。
- `+` / `-`: **同時ボール数（負荷）を増減**。性能比較の主軸。

## ルール / 数値（全エンジン共通）
- パドル: 幅 **96** × 高さ **18**。中心 y=**510** 固定。移動速度 **600 px/s**。左右画面内にクランプ。
- ボール: 半径 **8**、速さ **380 px/s**（一定）。発射時は上方向へランダム角（左右 ±60° 以内）。
- 反射:
  - 左右の壁・天井で反射。
  - パドルで反射（常に上方向へ）。**当たった位置のパドル中心からのオフセットで反射角を変える**
    （端ほど横に鋭く。速さは一定に正規化）。
  - ボールが下端（y > 540 + r）を抜けたら**ロスト → パドル上から再発射**して同時ボール数を維持
    （負荷を一定に保つためゲームオーバーにしない）。`Lost` をカウント。
- ブロック: グリッド配置。**15列 × 9行 = 135個**。ブロック 56x20 + 間隔4、上オフセット 60。
  - 行ごとに耐久（HP）。上3行=HP3 / 中3行=HP2 / 下3行=HP1。
  - 表示色は HP で変化（HP3=赤, HP2=橙, HP1=緑）。画像 `brick.png` は明色1枚を**HP色で tint（乗算）** して使う。
  - ボールが当たると HP-1、HP0 で破壊し **Score +10**。破壊時に `hit_spark` を一瞬表示（任意）。
- ボール×ブロック当たり判定: **AABB（矩形）×円（最近点）**。当たった面でボール速度を反転（左右面=vx反転 / 上下面=vy反転）。
  - 1ボールにつき1フレーム1ブロックまで（最初の命中で break）。
- **全ブロック破壊で盤面を再生成**（ベンチを継続させるため）。
- **ボール同士の衝突は行わない**（性能/簡潔さのため省略。READMEに明記）。
- 物理エンジンは使わず、自前で位置更新・反射・円/矩形判定を行う。

## 同時ボール数（負荷）
- 初期 **3** 個、`+`/`-` で **±5**（下限1・上限 **500**）。

## アセット（`../assets/` から読み込み）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `paddle.png` | パドル | 96x24 |
| `ball.png` | ボール | 16x16 |
| `brick.png` | ブロック(明色, HPで tint) | 64x24 |
| `hit_spark.png` | 破壊エフェクト | 32x32 |
| `bg_breakout.png` | 背景(タイル) | 512x512 |

- **画像が無い場合は単色図形でフォールバック**（パドル=白角丸, ボール=白丸, ブロック=HP色矩形[赤/橙/緑], spark=黄バースト, 背景=暗色）。

## 共通HUD（必須・性能比較の要）
画面左上にオーバーレイ表示:
- `FPS`（実測, 移動平均）
- `Objects`（ボール + 残ブロック + エフェクトの合計数）
- `Score`
- `Balls`（現在の同時ボール数 / 設定値）
- `Bricks`（残ブロック数）
