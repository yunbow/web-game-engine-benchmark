# テーマ2 共通仕様 ― トップダウン・サバイバー

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 見下ろし型。自機を操作し、四方から群がる大量の敵を**自動攻撃**で殲滅し生存時間を稼ぐ。
- **目的**: 性能比較。**数百体の敵の群れ更新＋当たり判定**で実用的な性能限界を測る。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。地面は `ground_tile.png` のタイル敷き詰め（無ければ単色）。
- カメラは自機追従（自機が概ね中央）。ワールドは無限スクロール扱いでよい。

## 操作
- 移動: WASD / 矢印（8方向、速度 180 px/s）。
- 攻撃: **オート**。最も近い敵へ 400ms ごとに弾を発射（弾速 350 px/s, 命中で敵HP-1）。
- `+` / `-`: 敵スポーン数（負荷）増減。

## ルール / 数値（全エンジン共通）
- 敵: 画面外周からスポーンし自機へ直進（速度 60〜90 px/s）。接触で自機HP-1（無敵0.5s）。
- 敵HP=1（小）/3（大）。撃破で `xp_gem` ドロップ、自機が触れて取得→Killカウント。
- 同時敵数: 初期150体、`+`/`-`で±50（上限1000）。**自動で徐々に増加**（10秒ごと+25）。
- 自機HP初期5。0で `GAME OVER` 表示＆リスタート可。
- 敵同士は重なってOK（簡易）。当たり判定は円。

## アセット（`../assets/` から読み込み）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `player.png` | 自機（上向き/正面） | 48x48 |
| `enemy_bat.png` | 小型敵(速い) | 32x32 |
| `enemy_zombie.png` | 大型敵(遅い/HP高) | 40x40 |
| `projectile.png` | 自動攻撃弾 | 24x24 |
| `xp_gem.png` | 経験値ジェム | 16x16 |
| `ground_tile.png` | 地面(タイル) | 64x64 |

- **画像が無い場合は単色図形でフォールバック**（自機=白丸, bat=紫丸, zombie=緑丸, 弾=黄, gem=水色菱形）。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Enemies`（生存敵数）/ `Objects`（敵+弾+gem合計）
- `Time`（生存秒）/ `Kills` / `HP`
