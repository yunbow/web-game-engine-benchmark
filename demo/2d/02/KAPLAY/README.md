# トップダウン・サバイバー — KAPLAY 版

テーマ2共通仕様（トップダウン・サバイバー・性能比較用）の **KAPLAY** 実装。
画像アセットが無くても単色図形フォールバックで必ず起動します。

使用バージョン: **KAPLAY 3001.0.19**（CDN: `unpkg.com/kaplay@3001.0.19/dist/kaplay.js`）。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`2/`)をルートに HTTP サーバを立てて開く。

```bash
cd 2
python -m http.server 8000
# → http://localhost:8000/KAPLAY/
```
`file://` 直開きは画像が CORS で読めない（図形フォールバックでは起動する）。

## 操作
- 移動: WASD / 矢印キー（8方向・速度 180px/s・正規化）
- 攻撃: **オート**（400ms ごとに最も近い敵へ弾速 350px/s で発射、命中で敵HP-1）
- `+` / `-`（テンキー +/- も可）: 敵スポーン上限を ±50（上限1000）
- GAME OVER 後: `R` でリスタート

## 実装メモ
- `kaplay({ canvas, width:960, height:540, global:false })` で初期化。名前空間 `k.*` を明示利用。
- KAPLAY の座標系は **Y 下向き・原点左上 = 画面座標と一致**するため座標変換が不要（three/A-Frame と違い最も素直）。
- カメラ追従は `k.camPos(player.pos)` を毎フレーム設定し自機を中央に固定。ワールドは無限スクロール扱い。
- 地面タイルは `onDraw` で自機周辺をタイル境界グリッドにループ描画（画像があれば `drawSprite`、無ければ市松の `drawRect`）。1タイル=1エンティティにしない（数百体の敵と並ぶと破綻するため描画コール方式）。
- 敵・弾・gem は数百規模になり得るので自前配列 + `rm()`（末尾スワップ削除）で管理。スポーンは `cap` まで補充しつつ 1フレーム最大40体に分散。
- 敵タイプ: 3割を大型（zombie, HP3, r16）/ 7割を小型（bat, HP1, r12）。撃破で gem ドロップ、自機が触れて取得→Kill カウント。
- 自機HP初期5、接触で-1（無敵0.5s）、0で `GAME OVER`。10秒ごとに cap +25 の自動増加。
- **当たり判定は SPEC 準拠の自前円判定（平方距離比較）**。KAPLAY の `area()` は AABB/多角形なので比較条件を揃えるため不使用。
- HUD は他エンジンと同じ HTML オーバーレイ（`#hud`）に FPS / Enemies(+cap) / Objects / Time / Kills / HP を表示。

## Codex / AI コーディング所感
- ミニマル API で「シーン構築〜ループ〜入力〜カメラ追従」を**一発生成しやすい**。`camPos` で見下ろし追従が即書けるのは本テーマと好相性。
- 罠: ① 旧 **Kaboom** との API 差（`kaplay()` 名称・コンポーネント名）で記憶が混ざる。② キー名が独自（`"left"`, `"kpadd"`, `"minus"`）で `e.code` ではない。③ 地面を 1タイル=1エンティティで敷くと数百体の敵更新と重なって FPS を落とすため、`onDraw` 直描画に倒す方針は明示指示が要る。④ `area()`/`onCollide` を使うと判定方式が他エンジンと変わり比較がブレるため、意図的に自前円判定にした。
