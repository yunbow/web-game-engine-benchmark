# テーマ11 2Dダイナミックライティング / 影 ― Babylon.js 版

暗いトップダウンの部屋を、プレイヤーの灯りと多数の動的な色付き光源が照らし、矩形の柱（オクルーダ）が**影**を落とす
ライティングベンチ。**多数の動的光源 ＋ ライトマップ合成（加算→乗算）＋ 影生成**を同時に回し、
「同時光源数」と「影 ON/OFF の描画コスト差」を測る。

## 起動方法

```bash
# このフォルダ (11/Babylon.js/) で
python -m http.server 8000
# → http://localhost:8000/ をブラウザで開く
```

`../assets/` が空でもフォールバック描画で必ず起動します。

## 使用バージョン / CDN

- Babylon.js（CDN: `https://cdn.babylonjs.com/babylon.js`）
- 追加ライブラリなし。`index.html` + `game.js` のみ。

## 操作

| キー | 動作 |
|---|---|
| `W`/`A`/`S`/`D` または 矢印 | プレイヤー（＝白色光源 半径240）を移動（220 px/s, 壁/柱で停止）|
| `+` / `-` | 動的光源の数を 6 ずつ増減（性能比較の主軸）|
| `L` | 影 ON / OFF 切替 |
| `R` | リセット |

## 仕様数値（全エンジン共通）

| 項目 | 値 |
|---|---|
| キャンバス / タイル | 960 x 540 / 32px（部屋 30 x 17）|
| ambient | 0.10 |
| プレイヤー光源 半径 / 速度 | 240 / 220 px/s |
| 動的光源 半径 / 速度 | 160 / ~120 px/s 相当の決定的軌道 |
| オクルーダ | 矩形・約16個 + 外周壁 |
| 動的光源数 | 初期 12・±6・下限 0・上限 120 |

## HUD（HTML `#hud` overlay・左上固定）

- `FPS`（実測・移動平均）/ `Lights`（現在 / 上限・＋プレイヤー）/ `Occluders` / `Shadows`(ON/OFF) /
  `Mode`(`Lightmap(canvas)`) / `Ambient`(0.10) + 操作ヒント。

## 実装メモ

### なぜ「Canvas2D 合成 → DynamicTexture で提示」なのか（このテーマの肝）

Babylon は本来 3D エンジンで、ライト/影機構（`PointLight` / `ShadowGenerator`）は**3D メッシュ＋深度マップ前提**。
本テーマが求める「放射状フォールオフ × 多数の色付き点光源 × 矩形オクルーダのハードシャドウ × 加算→乗算のライトマップ」を
`ShadowGenerator` で再現しようとすると、平面メッシュ化・正射影シャドウ・ライト数上限などの制約が多く、
**他の JS エンジン（自前ライトマップ）と比較対象がズレてしまう**。

そこで本実装は、SPEC が JS 系標準として挙げる**ライトマップ方式**をオフスクリーンの **2D Canvas**（Canvas2D 合成）で
「正直に」実装し、完成した1枚を Babylon の **`DynamicTexture`** として全画面の**正射影 Plane** に貼って**提示（present）だけ**を
Babylon に任せる。

1. **light バッファ**: 全光源を加算合成（`globalCompositeOperation='lighter'`）で積む。影 ON 時は光源ごとに矩形オクルーダの
   シルエットを**黒**の影ポリゴンで塗って光を削る（ハードシャドウ）。下地に `ambient` を足す。
2. **scene バッファ**: 暗い床タイル＋柱を描く。
3. scene に light を **`multiply`** 合成 → 照らされた所だけ見え、影/未照は沈む。
4. その1枚を `DynamicTexture` 経由で Babylon の全画面 Plane に提示。

→ ライティングは honest な 2D 合成、**Babylon は presenter**。この構成上の選択自体が「3Dエンジンで2Dライティングを
どう扱うか」という比較ポイントになる。

### カメラ / 座標系

- 正射影（Orthographic）カメラで画面座標 (0,0)=左上 / (960,540)=右下 を再現。Plane に DynamicTexture を貼って覆う。
- 柱配置・光源軌道・色は `mulberry32` の決定的乱数（`Math.random` 不使用）。

### フォールバック

- 画像があれば床/柱/グロー/プレイヤーに使い、無ければ Canvas2D の図形（放射グラデ等）でフォールバック。画像ゼロでも点灯。

## Codex 生成所感（3Dエンジンで2Dライティングを書く所感・罠）

- **正攻法（ShadowGenerator）に引きずられないことが最大の判断**。Babylon の知識でそのまま書くと、つい 3D ライト＋影に
  寄せたくなるが、それでは「2D の影込みライトマップ合成コスト」という比較軸からズレる。本テーマは
  **Canvas2D 合成を本体、Babylon を提示層**と割り切ったことで、他の JS 系（Phaser/Pixi/LittleJS）と同じ土俵に乗せられた。
- **`DynamicTexture` は2Dの canvas をそのまま GPU テクスチャ化できる**ので、「2Dで作って3Dで貼る」橋渡しが容易。
  毎フレーム `update()` するコストはあるが、ライトマップ自体が1枚なので転送は1回で済む。
- **罠は y 軸と Plane のスケール合わせ**。正射影窓と Plane のサイズ・UV 向きを 960x540 に正しく合わせないと、
  ライトマップが反転/ズレる。ortho 設定は他テーマ（5/6 等）の Babylon 実装と同じ流儀で固めた。
- 総じて Babylon は「2Dを描く」こと自体は presenter 役なら軽いが、**2Dライティングの本質的な計算はエンジンの外（Canvas）に
  出る**ため、このテーマでの Babylon の FPS は実質「Canvas2D 合成の速さ＋テクスチャ転送」を測ることになる。
  Godot のネイティブ `PointLight2D`（エンジン内蔵のライト/影）との対比が分かりやすい。
