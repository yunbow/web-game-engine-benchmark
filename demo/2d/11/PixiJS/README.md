# テーマ11 2Dダイナミックライティング / 影 ― PixiJS v8 版

暗いトップダウンの部屋を、プレイヤーの灯りと多数の動的な色付き光源が照らし、矩形の柱（オクルーダ）が**影**を落とす
ライティングベンチ。**多数の動的光源 ＋ ライトマップ合成（加算→乗算）＋ 影生成**を同時に回し、
「同時光源数」と「影 ON/OFF の描画コスト差」を測る。テーマ8（加算ブレンドのパーティクル発光）とは別軸の、
**光のボリューム・遮蔽影・ブレンド経路**の比較。

## 起動方法

```bash
# このフォルダ (11/PixiJS/) で
python -m http.server 8000
# → http://localhost:8000/ をブラウザで開く
```

`../assets/` が空でもフォールバック描画で必ず起動します（`file://` 直開きは画像が CORS で読めないため HTTP 推奨）。

## 使用バージョン / CDN

- PixiJS **v8**（CDN: `https://cdn.jsdelivr.net/npm/pixi.js@8/dist/pixi.min.js`）
- 追加ライブラリなし。ライティングは**自前のライトマップ合成**（`RenderTexture` ＋ ブレンドモード）。
- v8 仕様に準拠: `await app.init(...)` / `app.canvas` / 新 Graphics API（`g.rect().fill()` / `g.circle()`）/ `blendMode` は文字列（`'add'` / `'multiply'`）。

## 操作

| キー | 動作 |
|---|---|
| `W`/`A`/`S`/`D` または 矢印 | プレイヤー（＝白色光源 半径240）を移動（220 px/s, 壁/柱で停止）|
| `+` / `-` | 動的光源の数を 6 ずつ増減（性能比較の主軸）|
| `L` | 影 ON / OFF 切替 |
| `R` | リセット |

## 仕様数値（全エンジン共通）

| 項目 | 値 |
|---|---|
| キャンバス / タイル | 960 x 540 / 32px（部屋 30 x 17）|
| ambient | 0.10 |
| プレイヤー光源 半径 / 速度 | 240 / 220 px/s |
| 動的光源 半径 / 速度 | 160 / ~120 px/s 相当の決定的軌道 |
| オクルーダ | 矩形・約16個 + 外周壁 |
| 動的光源数 | 初期 12・±6・下限 0・上限 120 |

## 実装メモ

### ライトマップ合成（描画の核・v8）

PixiJS は描画ライブラリのため、ライティングは自前のオフスクリーン合成で組む。

1. **シーン**（暗い床タイル＋柱）を stage に普通に描く。
2. **ライト RT**（`lightRT` = `RenderTexture.create(...)`）に、`ambient` の下地を引いた後、各光源の寄与を
   **`blendMode:'add'`** で積む。光源1個ぶんは:
   - プールした**スクラッチ RT** に放射グロー（`light_glow` テクスチャ／フォールバックは生成した放射グラデ）を描く。
   - **影 ON** なら、その光源から見た矩形オクルーダの**シルエット辺**を遠方（`SHADOW_PROJECT=2000`）へ延長した
     影ポリゴンを `Graphics` で**黒**に塗って光を削る。
   - スクラッチを `lightRT` へ加算。
3. `lightRT` を `Sprite` 化し **`blendMode:'multiply'`** でシーンへ重ねる。
- 描画指示は **`app.renderer.render({ container, target: renderTexture })`** を使う（v8 のシグネチャ）。

→ 光源数ぶんのスクラッチ RT 往復＋影ポリゴン生成が「影 ON」のコスト。`L` で影を切ると軽くなる。

### 決定的生成 / フォールバック

- 柱配置・光源軌道・色は `mulberry32` の決定的乱数（`Math.random` 不使用）。
- `PIXI.Assets.load` の失敗を `catch` して `null` 化し、放射グラデ等の図形フォールバックへ切替。画像ゼロでも点灯する。

## Codex 生成所感（v8 でライトマップを組む所感・罠）

- **v8 の `renderer.render({ target })` と文字列 `blendMode` を押さえれば素直**。`'add'` で光を積み `'multiply'` で
  乗算合成、という2パスがそのまま書ける。Graphics の新 API（`circle().fill()` / `poly()`）で放射グローのマスクや
  影ポリゴンも書きやすい。
- **最大の罠は RenderTexture の扱い**。`RenderTexture.create` で作ったテクスチャに描くには `renderer.render` の
  `target` を使う必要があり、`app.stage` に足して自動描画される通常スプライトとは経路が違う。ここを取り違えると
  「ライトが画面に直接出てしまう / 何も映らない」になりやすく、v8 のシグネチャ明示が成功の鍵だった。
- **スクラッチ RT はプール必須**。光源ごとに作ると即破綻するため、1枚を clear して使い回す設計を明示。
  これは Phaser 版とまったく同じ勘所で、描画ライブラリ系の共通課題。
- PixiJS には `@pixi/lights` 等もあるが、本テーマは「影込みライトマップ合成のコスト」を全エンジンで揃えるため
  自前実装にした。結果、Godot のネイティブ `PointLight2D` との対比（自前ブレンド vs エンジン内蔵）がきれいに出る。
