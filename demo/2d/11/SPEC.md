# テーマ11 共通仕様 ― 2Dダイナミックライティング / 影（ライトマップ × 多光源 × 影）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 暗いトップダウンの部屋。プレイヤーが灯りを持って歩き、決定的に動く多数の色付き光源が部屋を照らす。柱（オクルーダ）が**影を落とす**。
- **目的**: 性能比較。**多数の動的光源 ＋ ライトマップ合成（加算→乗算）＋ オクルーダによる影生成**を同時に回した性能を測る。
  - テーマ8の「加算ブレンドのパーティクル（発光）」とは別軸の、**光のボリューム・遮蔽影・ライト/シェーダ経路**を比較する。
- 性能比較の核は「**同時に存在する動的光源数**」と「**影 ON/OFF の描画コスト差**」。

## ライティングのモデル（全エンジン共通の考え方）
- 画面は**暗い基準（ambient = 0.10）**から始まる。各光源は中心が明るく外周が 0 に落ちる**放射状フォールオフ**を持つ。
- **ライトマップ方式（JS系の標準実装）**:
  1. オフスクリーンの**ライトバッファ**に、全光源の寄与を**加算合成**で積む（ambient を下地に足す）。
  2. 各光源の寄与には**影**を反映する（後述。光源ごとに、オクルーダの背後を暗くする）。
  3. 完成したライトバッファを、描画済みのシーン（床・柱）に**乗算合成（multiply）**で重ねる。→ 照らされた所だけ見え、影/未照は暗く沈む。
- **Godot はネイティブ機構を使う**: `CanvasModulate`（暗い ambient）＋ `PointLight2D`（グラデーションテクスチャ）＋ `LightOccluder2D`（影 `shadow_enabled`）。
  これ自体が比較対象（「ネイティブ2Dライト」vs「自前ライトマップ＋ブレンド」）。各 README に使った機構を必ず記載する。
- **法線マップ**は任意（対応エンジンのみ）。必須ではない。使った場合は HUD `Mode` に明記。

## 影（オクルーダ）の生成
- オクルーダは**軸平行の矩形（柱/壁）**のみ。決定的に配置。
- 影生成（JS系の推奨アルゴリズム / ハードシャドウ）:
  - 光源ごとに**スクラッチ用ライトバッファ**を用意（プールして使い回す）。クリア → その光源の放射グラデを描く。
  - 各矩形オクルーダについて、光源から見た**シルエット辺**（光源側を向いていない辺）の端点を、光源から遠ざかる方向へ
    光半径を超えて延長し、**影ポリゴン**（四角形）を作る。これを**黒（=光ゼロ）でスクラッチバッファに描いて光を削る**。
  - 仕上げにスクラッチバッファ（=その光源の影込み寄与）を全体ライトバッファへ**加算**。
  - ＝ 光源数ぶんのバッファ往復が発生する（これが「影 ON」のコスト＝比較対象）。
  - **影が技術的に困難なエンジンは、近似実装でもよい**（オクルーダ背後が視認できる程度に暗くなり、`L` で ON/OFF 差が出ること）。
    採用した方式と限界を README に明記する。Godot は `LightOccluder2D` でネイティブに行う。
- `L` キーで**影 ON/OFF を切替**（性能比較の主軸の一つ。影なし＝放射光のみの加算で軽い）。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。タイル **32x32** の暗い床。
- 部屋: **30 x 17 タイル**相当（960x544）。外周は壁、内部に**矩形オクルーダ（柱）を決定的に約16個**配置（固定シード）。
- 時間はデルタタイム基準。

## エンティティ
- **プレイヤー光源**: プレイヤーアバターが持つ白色光（半径 **240**）。`WASD`/矢印で移動（**220 px/s**）、壁/柱で停止（簡易AABB）。
- **動的光源（負荷の主役）**: 決定的に動く色付き点光源（半径 **160**, ランダム色）。
  - 各光源は固定シードで決まる軌道（バウンド/円運動）を**自動**で動く（無入力でもベンチが回る）。柱は通り抜けてよい（影は落とす）。
- **オクルーダ**: 矩形の柱。動かない。影を落とす（プレイヤー光源＋全動的光源の双方に対して）。

## 操作
- 移動: `WASD` / 矢印（プレイヤー光源を移動）。
- `+` / `-`: **動的光源の数（負荷）を増減**。性能比較の主軸。
- `L`: 影 ON/OFF 切替。
- `R`: リセット。

## ルール / 数値（全エンジン共通）
- ambient = **0.10**（下地の明るさ）。光は加算で積み、最終的に**乗算**でシーンへ。光の合計は 1.0 でクリップしてよい。
- 光源フォールオフ: 中心 1.0 → 半径で 0.0（`smoothstep` 等、滑らかに）。色は加法混色（重なるほど明るく/色が混ざる）。
- 動的光源: 初期 **12** 個、半径 160、速度 ~120 px/s 相当の決定的軌道。色は決定的PRNGで割当。
- これはライティングデモのため勝敗は無い（描画統計をHUDに出す）。
- 決定的生成（固定シード）。`Math.random` 不使用。

## 光源数（負荷）
- 初期 **12** 個、`+`/`-` で **±6**（下限 **0**・上限 **120**）。プレイヤー光源は常に1つ別枠。
  - 増やすほど「加算する光源数」と（影ONなら）「光源ごとのバッファ往復・影ポリゴン生成」が増える。
    Godot のネイティブライト（光源ごとにシャドウマップ）と、JS系の自前ライトマップで `FPS` の落ち方の差が見える。

## アセット（`../assets/` から読み込み）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `tile_floor.png` | 暗い床（タイル可） | 64x64 |
| `pillar.png` | 柱（オクルーダ） | 64x64 |
| `light_glow.png` | 光源の放射グロー（中心白→外周透明・加算前提） | 256x256 |
| `player_lamp.png` | プレイヤー（ランタン持ち） | 32x48 |

- **画像が無い場合は図形フォールバック**（床=暗い灰タイル / 柱=灰の矩形＋縁 / 光=コードで放射グラデ生成 / プレイヤー=小さな人型＋灯り）。
  - `light_glow.png` が無くても、各エンジンで放射グラデーション（Canvas/DynamicTexture/Graphics 等）を生成して必ず点灯する。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Lights`（動的光源数 現在 / 上限。＋プレイヤー光源は別表記）
- `Occluders`（影を落とす柱の数）
- `Shadows`（ON / OFF）
- `Mode`（使ったライティング機構: 例 "Lightmap(blend)" / "PointLight2D" / "+normalmap" 等）
- `Ambient`（下地の明るさ）
- 画面下に操作ヒント行（WASD=移動 / +/-=光源数 / L=影 / R=リセット）。
