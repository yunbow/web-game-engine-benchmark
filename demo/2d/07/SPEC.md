# テーマ7 共通仕様 ― 物理パズル（剛体物理エンジン統合 / 投擲物理）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 積み上げた箱（剛体）の山に発射体（弾）をぶつけて崩す、投擲物理の物理パズル。
- **目的**: 性能比較。**本物の2D剛体物理エンジンを“使った”ときの統合相性と剛体数スケール**を測る。
  - テーマ4（ブロック崩し）・テーマ5（横スクロール）は「物理エンジン**不使用**・自前判定」が主題だった。
    本テーマはその**対**として、各エンジンが推奨/内蔵する剛体物理エンジンを**積極的に使う**ときの実装難度・性能を比較する。
- 性能比較の核は「**シーン内の剛体数**」と「**覚醒（active/awake）している剛体数**」。

## 使用する物理エンジン（エンジンごとに“素直な選択”を使う ＝ これ自体が比較対象）
- **Phaser4**: 内蔵の **Matter.js**（`physics: { default: 'matter' }`）。
- **PixiJS**: 描画専用のため **matter.js を CDN 併用**（`matter-js`）。Pixi スプライトを Matter ボディに毎フレーム同期。
- **Babylon.js**: 3Dエンジンだが本テーマは2D。**matter.js を CDN 併用**し、正射影カメラ＋2Dスプライト相当で表示（Babylon物理は3D前提のため2D剛体は matter で扱う）。
- **LittleJS**: 公式 **Box2D プラグイン（`box2d.js` / `littlejs.box2d`）**を使用。利用不可な場合は matter.js を CDN 併用してよい（README に明記）。
- **Godot**: 内蔵の **RigidBody2D / 物理サーバ**（`StaticBody2D` の地面 + `RigidBody2D` の箱）。
- いずれも「重力・接触・スタック・反発」を**物理エンジンに委譲**する（テーマ4/5のような自前AABBは使わない）。
- どの物理エンジンを使ったか・統合方法・つまずきを各 README の「実装メモ／Codex生成所感」に必ず記載する。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。座標は左上原点・px・画面下が地面（各エンジンのY軸規約は内部で吸収）。
- 重力: **下向き 9.8 相当**（物理エンジンの単位に合わせ、見た目で「箱が約 1〜1.5s で落ち着く」程度に調整。px系なら ~1000 px/s²）。
- 地面: 画面下端から **上 48px** に水平な静的床（`STATIC`）。左右にも静的な壁（場外流出防止）。
- ワールドは1画面固定（スクロールなし）。

## 構造物（決定的に生成）
- 画面右側に箱を**ピラミッド状/格子状に積む**。1箱 **34x34 px**、密度は負荷値（箱数）に応じて段数・列数を決定的に決める。
- 箱は剛体（`DYNAMIC`）。床・壁は静的。初期配置・初期姿勢は**固定シード**で決定的（Math.random不使用）。
- 任意で、頂上に「ターゲット（壊すと加点する特別ブロック, 色違い）」を決定的に数個配置。

## 操作（発射体 = projectile）
- **左ドラッグ＆リリース**（スリングショット）: 画面左の発射台から、ドラッグ方向と距離に比例した初速で発射体を撃つ。
  - ドラッグ無しの**クリックのみ**でも、クリック地点へ向けた固定初速で発射してよい（簡易操作）。
- 発射体は剛体の円（半径 **12**）。床・箱と物理衝突して山を崩す。画面外or静止後しばらくで消滅。
- `Space`: 自動発射トグル（**オートショット**）。ON の間、**0.8s 間隔で発射台から決定的な角度/初速のショットを自動発射**
  し続ける（マウス無しでベンチを回せるようにする）。
- `+` / `-`: **シーンの箱（剛体）数を増減**。性能比較の主軸。
- `R`: 構造物・スコアをリセット（決定的に再構築）。

## ルール / 数値（全エンジン共通）
- 箱を発射体で崩し、**箱が一定速度以上で他と衝突 or 落下**したら “崩した” と判定して **Score +10**（重複加点を避けるため箱ごとに1回）。
  - 簡易実装可: 箱の重心が初期位置から **64px 以上** 移動したら「崩した」とみなして加点（1回）。ターゲット箱は **+50**。
- 場外（床より下/壁の外）に出た剛体は**スリープ後に除去**してもよい（剛体数の暴走防止）。除去は負荷値の維持に影響しない。
- 発射体は最大 **同時 8 発**までプール。古いものから消す。
- 物理は完全に物理エンジン任せ（接触・反発・摩擦・スリープ）。

## 剛体数（負荷）
- 箱の初期数 **60** 個、`+`/`-` で **±20**（下限 **20**・上限 **600**）。
  - 増やすほど剛体接触解決（broad/narrow phase, 拘束ソルバ）の負荷が増える。`Active bodies`（覚醒剛体数）が
    安定後に落ち着く（スリープ）か張り付くかでエンジンのソルバ/スリープ実装差が見える。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Bodies`（剛体総数 = 箱 + 発射体 + 床/壁を含めた合計、または「箱数 / 設定値」を主に表示）
- `Active`（覚醒している剛体数。スリープ中は除く）
- `Shots`（発射済み/生存中の発射体数）
- `Score` / `Engine`（使用した物理エンジン名: Matter / Box2D / RigidBody2D）
- 画面下に操作ヒント行（ドラッグ=発射 / Space=オート / +/-=箱数 / R=再構築）。
