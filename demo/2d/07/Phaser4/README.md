# テーマ7 物理パズル (投擲物理) ― Phaser 4 版 / Matter.js

積み上げた箱(剛体)の山に、スリングショットから発射体をぶつけて崩す 投擲物理の
物理パズル。テーマ4/5が「物理エンジン**非使用**・自前 AABB」だったのに対し、本テーマは
その**対**として、Phaser 4 が**内蔵する 2D 剛体物理エンジン Matter.js を積極的に使う**ときの
統合相性と剛体数スケールを測る比較用サンプル。重力・接触・スタック・反発・スリープは
すべて Matter.js に委譲する。

## 起動方法

ブラウザのセキュリティ制約(file:// での CORS / アセット読込)を避けるため、
簡易 HTTP サーバ経由での起動を推奨します。

```bash
# このフォルダ (7/Phaser4/) で
python -m http.server 8000
# → http://localhost:8000/ をブラウザで開く
```

または任意の静的サーバ(`npx serve` など)でも可。
アセット画像が無くてもフォールバック描画で必ず起動します。

## 使用バージョン

- Phaser **4**(CDN: `https://cdn.jsdelivr.net/npm/phaser@4/dist/phaser.min.js`)
- 物理エンジンは **Phaser 内蔵の Matter.js**(`physics: { default: 'matter' }`)。
  **追加の物理 CDN は不要**(Pixi/Babylon と異なり Phaser は Matter を同梱)。
- `index.html` + `game.js` のみ。

## 操作

| 操作 | 動作 |
|---|---|
| 左ドラッグ & リリース | 引いた方向と逆向き(パチンコ)に、距離比例の初速で発射 |
| クリックのみ(ドラッグ無し) | クリック地点へ向けた固定初速で発射 |
| `Space` | オートショット トグル(0.8s 間隔で決定的角度/初速を自動発射) |
| `+` / `-`(テンキー可) | 箱(剛体)数を 20 個ずつ増減して再構築 |
| `R` | 構造物・スコアを決定的に再構築 |

- 発射体は剛体の円(半径 12)。床・箱と物理衝突して山を崩す。
- 箱の重心が初期位置から **64px 以上**移動したら「崩した」とみなし加点(箱ごとに 1 回)。
  通常箱 **+10** / ターゲット箱(橙) **+50**。
- 発射体は**同時最大 8 発**のプール。満杯時は最古を除去。場外/静止/寿命超過で消滅。
- 場外へ出た箱はスリープ後に除去(剛体数の暴走防止。設定値=負荷値には影響しない)。

## HUD

`setScrollFactor(0)` で固定し、左上に常時表示。

- `FPS`(実測・移動平均 30 サンプル)
- `Bodies`(現存する箱数 / 設定値)
- `Active`(覚醒している剛体数 = `body.isSleeping === false` の箱+発射体)
- `Shots`(生存中の発射体数 / 上限 8、累計発射数)
- `Score` / `Engine: Matter`
- 画面下に操作ヒント行(ドラッグ / Space / +- / R)。

## 実装メモ ― Matter.js 統合方法

### 物理エンジンの選択

- Phaser 4 は **Matter.js を同梱**しており、`config.physics.default = 'matter'` を指定する
  だけで利用できる。Pixi/Babylon が描画専用で `matter-js` を CDN 併用するのに対し、
  Phaser は**追加 CDN 不要**で内蔵物理を使えるのがこのテーマでの“素直な選択”。

### ボディとスプライトの同期

- 箱・発射体は **「描画用スプライト(`this.add.image`)」と「物理ボディ(`this.matter.add.rectangle/circle`)」を
  別々に作り、毎フレーム手動で同期**する方式を採った。
  `update()` で `sprite.setPosition(body.position.x, body.position.y)` と
  `sprite.setRotation(body.angle)` を行い、描画を物理に追従させる。
  - `this.matter.add.sprite(...)`(ボディ付きスプライト)も使えるが、本サンプルは
    「ボディ配列を自前管理して場外除去・加点判定・Active カウントを回す」構造のため、
    ボディとスプライトを分離管理するほうが見通しが良かった。
- ボディ生成は `this.matter.add.rectangle(cx, cy, w, h, opts)` / `this.matter.add.circle(...)`。
  これらは Matter ボディを生成して**そのままワールドへ追加**し、ボディ参照を返す。
  除去は `this.matter.world.remove(body)`、初速付与は `this.matter.body.setVelocity(body, {x, y})`。

### 静的ボディ(地面・壁)

- 床・左右壁・天井は `isStatic: true` の矩形ボディ。重力・接触で動かず、箱/弾を受け止める。
- 床は上面が `GROUND_TOP (= 540 - 48 = 492)` に来るよう、厚みの半分ぶん下げて中心を配置。
- 左右壁は中心を画面外側に置き、内側の面だけが効くようにして場外流出を防ぐ。
- 静的ボディは常にスリープ扱いのため、後述の `Active` 数には含めない(動的剛体のみカウント)。

### スリープと Active 数(性能比較の核)

- `config.physics.matter.enableSleeping = true` を有効化。静止した動的ボディは Matter が
  自動的にスリープ(`body.isSleeping = true`)させ、ソルバ対象から外れて負荷が下がる。
- HUD の **`Active`** は `body.isSleeping === false` の動的ボディ(箱+発射体)を毎フレーム
  数えたもの。発射直後やスタック崩壊中は Active が跳ね上がり、安定後にスリープして落ち着く
  ――この**落ち着き方/張り付き方**にエンジンのソルバ・スリープ実装差が出る。
- 各箱には `sleepThreshold` を与え、静止判定をやや早めて Active が安定しやすいよう調整した。

### 重力チューニング

- Matter の重力は `gravity.y`(既定スケール)で与える。Matter は内部固定タイムステップで
  `y = 1.0` が地球重力相当になるよう調整されており、本サンプルでもそのまま `1.0` を採用。
  34px 箱・上記密度/摩擦/反発の組み合わせで、初期スタックが**約 1〜1.5s で落ち着く**ことを確認。
- 箱は `friction: 0.55 / frictionStatic: 0.8 / restitution: 0.05 / density: 0.0016`、
  発射体は `friction: 0.2 / restitution: 0.35 / density: 0.004`。発射体をやや高密度・高反発に
  することで「ぶつけて崩す」手応えを出した。スタック安定のため
  `positionIterations: 8 / velocityIterations: 6` とソルバ反復を既定より増やしている。

### 構造物の決定的生成(シード固定)

- 箱の積み方・ターゲット位置・オートショットの角度/初速は `mulberry32` 製の決定的疑似乱数で
  生成し、`Math.random` は不使用。`R` や `+`/`-` で**完全に同じ配置を再現**できる。
- 配置はピラミッド状(下段ほど広い)を基本に、箱数(負荷値)から段数・列数を決定的に決める。
  各箱に微小な x ジッタを与えて格子の不自然さを緩和。ターゲット箱(色違い・加点)は上段から
  決定的に 2〜5 個選ぶ(箱数に応じて増える)。
- オートショット用 PRNG は再構築時に巻き戻すため、`Space` での自動発射列も毎回同一になる。

### 発射(スリングショット)

- `pointerdown` で起点を記録し、`pointerup` で「起点 − リリース点」ベクトル(=引いた逆方向)に
  距離比例の初速を与えて発射。距離が極小ならクリック操作とみなし、クリック地点へ固定初速で発射。
- 初速は `MAX_LAUNCH_SPEED` でクランプ。発射体は最大 8 発のプールで、満杯なら最古を除去。
- `Space` のオートショットは 0.8s 間隔で右上方向中心に決定的に揺らした角度/初速を自動発射し、
  **マウス無しでベンチを回せる**ようにした(OFF→ON 直後は即発射)。

### アセット / フォールバック

- `../assets/`(= `7/assets/`)から SPEC のファイル名(box / box_target / ball / ground /
  slingshot / bg_sky)で読込。`BootScene` で `loaderror` を捕捉し、欠落キーは
  `Graphics.generateTexture` で焼いた図形にフォールバック
  (箱=木目茶矩形 / ターゲット=橙矩形 / 発射体=赤丸 / 地面=緑茶 / 発射台=灰 / 背景=空色)。
  **画像が 1 枚も無くても必ず起動する。**

## Codex 生成所感(内蔵 Matter.js 統合のしやすさ)

- **内蔵 Matter の手軽さ**: Phaser は Matter を同梱しているため、`config.physics` に
  `default: 'matter'` と重力を書くだけで剛体物理が立ち上がる。Pixi/Babylon のように
  `matter-js` を別 CDN で読み、Engine/World を自前で `Engine.update` する手間が一切無いのが
  最大の利点で、テーマ4/5の自前 AABB と比べても「重力・接触・スタック・反発」を完全に委譲できて
  実装量が劇的に減った。
- **ボディ⇔スプライト同期**: `this.matter.add.sprite` でボディ付きスプライトを作れば同期は自動だが、
  本サンプルは場外除去・加点・Active カウントのためボディを自前配列で管理したく、あえて
  ボディとスプライトを分離して `update()` で手動同期した。`body.position` / `body.angle` を写すだけで
  済み、Matter 側の API(`matter.add.rectangle/circle`, `matter.body.setVelocity`,
  `matter.world.remove`)も素直だった。
- **スリープと Active**: `enableSleeping` + `body.isSleeping` で「覚醒剛体数」がそのまま取れるのは
  性能比較と好相性。スタックが落ち着くと Active が綺麗に 0 近くまで落ちる挙動が観測でき、
  HUD にそのまま出せた。
- **チューニング**: 「箱が 1〜1.5s で落ち着く」感触は density/friction/restitution と
  ソルバ反復数の調整で出した。Matter は単位が px 直結で直感的だが、密度が小さすぎると
  スタックが滑り、反発が高すぎると弾むので、値の追い込みには多少の試行が要った。
- 総じて Phaser 4 は「内蔵物理で剛体パズルを最短で立ち上げる」用途に非常に強く、
  剛体数(箱数)を増やしたときの Active/FPS の振る舞いを比較する土台が短時間で整った。
