# テーマ1 共通仕様 ― 弾幕STG（縦スクロールシューティング）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 縦画面想定の縦スクロールSTG。自機は下部、敵は上から出現。
- **目的**: 性能比較。画面上の弾・敵を多数描画し、FPSと表示オブジェクト数を可視化する。

## 画面・基本設定
- キャンバス: **960 x 540**（16:9, 固定）。背景は宇宙（暗色 or `bg_space.png`）。
- 60FPS目標。デルタタイム基準で更新。

## 操作
- 移動: 矢印キー / WASD（自機は8方向移動、画面内クランプ）。
- 発射: 常時オート連射（150msごと）。
- `+` / `-`: 敵の同時出現数（負荷）を増減。**性能比較用の負荷調整**。

## ルール / 数値（全エンジン共通）
- 自機弾速: 600 px/s（上方向）。連射間隔150ms。
- 敵: 上端からランダムX出現、下方向へ 80〜140 px/s。一定間隔で自機方向へ弾（敵弾速 200 px/s）。
- 敵スポーン: 初期は同時最大40体。`+`/`-`で±10（上限300）。
- 当たり判定: 円判定。自機弾×敵、敵弾×自機、敵×自機。
- 被弾でHP-1（初期HP3）。敵撃破でスコア+10。
- 撃破時に explosion を一瞬表示（任意）。

## アセット（`../assets/` から読み込み）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `player_ship.png` | 自機（上向き） | 64x64 |
| `enemy_small.png` | 小型敵 | 48x48 |
| `enemy_big.png` | 大型敵 | 96x96 |
| `bullet_player.png` | 自機弾 | 16x24 |
| `bullet_enemy.png` | 敵弾 | 16x16 |
| `explosion.png` | 爆発 | 64x64 |
| `bg_space.png` | 背景(縦タイル) | 512x512 |

- **画像が無い場合は単色図形でフォールバック**して必ず起動すること（自機=水色三角、敵=赤丸、弾=黄丸 等）。

## 共通HUD（必須・性能比較の要）
画面左上にオーバーレイ表示:
- `FPS`（実測, 移動平均）
- `Objects`（画面上の弾+敵+エフェクトの合計数）
- `Score` / `HP`
- 現在の最大敵数設定

## 成果物
- JS系: `index.html`（＋必要なら `game.js`）。CDNでライブラリ読込。ダブルクリック/ローカルサーバで起動。
- Godot: プロジェクト一式（`project.godot`, `*.tscn`, `*.gd`）＋ HTML5 エクスポート手順を `README.md` に。
- 各フォルダに `README.md`（起動方法・実装メモ・Codex生成のしやすさ所感）。
