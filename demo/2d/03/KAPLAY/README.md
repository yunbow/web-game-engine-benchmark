# トップダウンRPG探索 — KAPLAY 版

テーマ3共通仕様（見下ろし型RPG探索・大マップスクロール描画の性能比較用）の **KAPLAY** 実装。
画像アセットが無くても単色タイル/図形フォールバックで必ず起動します。

使用バージョン: **KAPLAY 3001.0.19**（CDN: `unpkg.com/kaplay@3001.0.19/dist/kaplay.js`）。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`3/`)をルートに HTTP サーバを立てて開く。

```bash
cd 3
python -m http.server 8000
# → http://localhost:8000/KAPLAY/
```
`file://` 直開きは画像が CORS で読めない（図形フォールバックでは起動する）。

## 操作
- 移動: WASD / 矢印キー（4方向、速度 160px/s。壁/水/木に衝突）
- `Shift`: ダッシュ（2倍速）
- `+` / `-`（テンキー +/- も可）: 画面上のエンティティ（NPC+敵スライム）数を ±10

## 画面・基本設定
- キャンバス **960 x 540**（固定）、タイル **32x32**、マップ **100 x 100 タイル**（3200x3200px）。
- マップは固定シード mulberry32 で決定的生成（草/道/水/壁/木、外周は壁）。PixiJS 正準実装と同じ手順・同じ見た目。

## 実装メモ
- `kaplay({ canvas, width:960, height:540, global:false })` で初期化。名前空間 `k.*` を明示利用。
- KAPLAY の座標系は **Y 下向き・原点左上 = 画面座標と一致**するため、座標変換は不要。
- **カリング（本テーマの肝）**: KAPLAY の `camPos` は使わず、自前で `camX/camY` を計算し、可視範囲ぶん（`ceil(W/32)+2` × `ceil(H/32)+2` ≒ 32×19 枚）の**タイルスプライトプールを再利用**して `pos = tile*32 - cam` に毎フレーム再配置。**可視タイルのみ描画**する（PixiJS 正準実装と同一戦略）。
- 木（32x48 障害物）は別プールで地面（草）の上に重ね、`z = タイルY座標` で自機/エンティティと深度ソート。
- 衝突は SPEC 準拠の自前タイル矩形判定（軸分離移動）。スライム接触でノックバック。
- アセットは `loadSprite` を個別 try/catch、欠落分のみ図形（`rect`/`circle` + `color`／`onDraw`）にフォールバック。
- HUD は他エンジンと同じ HTML オーバーレイ（`#hud`）に **FPS / Tiles drawn（当該フレーム描画タイル数）/ Entities（NPC+敵+木）/ 自機タイル座標** を表示。

## Codex / AI コーディング所感
- ミニマル API で「シーン構築〜ループ〜入力」を一発生成しやすい。Kaboom 時代から訓練データもあり素直。
- 罠: ① 旧 **Kaboom** との API 差（`kaplay()`・コンポーネント名）で記憶が混ざる。② キー名が独自（`"left"`, `"kpadd"`, `"minus"`）で `e.code` ではない。③ **広大マップは `camPos` で全タイル add すると 1万枚で破綻**するため、本実装は「可視範囲だけ毎フレーム再配置するスプライトプール」を明示採用。AI を素直に走らせると全タイルを add しがちで、カリング方針の明示指示が要る。
