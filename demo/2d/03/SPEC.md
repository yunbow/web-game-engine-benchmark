# テーマ3 共通仕様 ― トップダウンRPG探索（マップ歩行）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 見下ろし型RPGの探索パート。タイルマップの世界を自機が歩き、壁で衝突、カメラ追従。
- **目的**: 性能比較。**大きなタイルマップ描画＋カメラスクロール**の効率を測る（RPG-Coboの題材に最も近い）。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。タイルサイズ **32x32**。
- マップ: **100 x 100 タイル**（=3200x3200px）。配列で定義（草/道/水/壁/木）。
  - 0=草, 1=道, 2=水(進入不可), 3=壁(進入不可), 4=木(進入不可)。
  - マップは決定的に生成（疑似ランダム＋外周は壁）。全エンジンで同じ見た目を目指す。
- カメラは自機追従。**可視範囲のタイルのみ描画**（カリング）すること。

## 操作
- 移動: WASD / 矢印（4方向、速度 160 px/s）。壁/水/木に衝突。
- `Shift`: ダッシュ（2倍速）。
- `+` / `-`: 画面内に配置する敵/NPC数（負荷）を増減。

## ルール / 数値（全エンジン共通）
- NPC/敵スライムをマップ上に配置（初期合計60体）。スライムは簡易ランダム徘徊。
- 自機がスライムに接触でノックバック（戦闘は簡易/任意）。
- アイテム（任意）。本筋は「広いマップを快適にスクロール描画できるか」。

## アセット（`../assets/` から読み込み）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `tile_grass.png` | 草(タイル) | 32x32 |
| `tile_path.png` | 道(タイル) | 32x32 |
| `tile_water.png` | 水(タイル) | 32x32 |
| `tile_wall.png` | 壁(タイル) | 32x32 |
| `tree.png` | 木(障害物) | 32x48 |
| `player.png` | 自機 | 32x32 |
| `npc.png` | NPC | 32x32 |
| `enemy_slime.png` | 敵スライム | 32x32 |

- **画像が無い場合は単色タイル/図形でフォールバック**（草=緑, 道=茶, 水=青, 壁=灰, 自機=白, NPC=黄, slime=緑丸）。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Tiles drawn`（そのフレームで描画したタイル数）/ `Entities`（NPC+敵+木）
- 自機タイル座標 `(x, y)`
