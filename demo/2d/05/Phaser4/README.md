# テーマ5 横スクロールアクション ― Phaser 4 版

横スクロールジャンプアクションの横スクロールジャンプアクション。200x17 タイル(6400x544px)の横長マップを
カメラ水平追従でスクロールし、**可視範囲のタイルのみ描画(カリング)** しながら、
**自前実装の重力・AABB タイル衝突** と **多数の敵更新** を同時に回して性能を測る比較用サンプル。
テーマ3の「大マップ描画」に重力物理と多数エンティティを足した題材。

## 起動方法

ブラウザのセキュリティ制約(file:// での CORS / アセット読込)を避けるため、
簡易 HTTP サーバ経由での起動を推奨します。

```bash
# このフォルダ (5/Phaser4/) で
python -m http.server 8000
# → http://localhost:8000/ をブラウザで開く
```

または任意の静的サーバ(`npx serve` など)でも可。
アセット画像が無くてもフォールバック描画で必ず起動します。

## 使用バージョン

- Phaser **4**(CDN: `https://cdn.jsdelivr.net/npm/phaser@4/dist/phaser.min.js`)
- 追加ライブラリなし。`index.html` + `game.js` のみ。
- **物理エンジン非使用**(Arcade 等を使わず、重力・AABB・反射をすべて自前実装)。

## 操作

| キー | 動作 |
|---|---|
| `←` / `→` または `A` / `D` | 左右移動 (歩き 180 px/s) |
| `Space` / `↑` / `W` | ジャンプ (接地時のみ・可変ジャンプ) |
| `Shift` | ダッシュ (288 px/s, ×1.6) |
| `+` / `-`(テンキー可) | 敵数(負荷)を 10 体ずつ増減 |

- 敵(クリボー風)を**踏む**と撃破(Score +100, スパーク表示, 自機は跳ねる)。
- 敵に**横から触れる**と被弾(HP-1, ノックバック, 無敵 1.0s)。
- コインに触れると取得(Coins++, Score +50)。
- 穴に落下 / HP0 でスポーン地点へ復帰(スコア・敵はベンチ継続のため保持)。

## HUD

`setScrollFactor(0)` でカメラ非追従に固定し、左上に常時表示。

- `FPS`(実測・移動平均 30 サンプル)
- `Tiles drawn`(そのフレームで描画したタイル数)/ `Entities`(敵 + コイン)
- 自機タイル座標 `(x, y)`
- `Score` / `Coins` / `HP`
- `Enemies`(現在の敵数 / 設定値)
- 画面下に操作ヒント行(移動 / ジャンプ / ダッシュ / +/-)。

## 実装メモ

### カリング(描画の核)

- カメラの `worldView`(ワールド座標の可視矩形)から可視タイル範囲 (x0..x1, y0..y1) を毎フレーム算出。
  範囲が前フレームから変化したときだけ Blitter を再構築し、可視タイルぶんの Bob のみ生成する。
- 地形種別(地面 / ブロック / 土管)ごとに **Blitter** を 1 枚ずつ用意(Blitter は単一テクスチャ +
  多数 Bob を高速バッチ描画できるためタイルに好適)。`Tiles drawn` に描画タイル数を表示。
  マップは 200x17=3400 タイルだが、常に可視ぶん(約 32x19 ＋余白 ≒ 数百タイル、空タイルは除外)しか描かない。
- コインも可視範囲のみ別 Blitter に再構築し、`Tiles drawn` とは別カウント(Entities 側)で管理。
- カメラは `setBounds(0,0,6400,544)` + `startFollow` で水平追従。`scrollX` は bounds により
  自動的に `[0, 6400-960]` にクランプされ、高さ 544≈540 のため縦スクロールはほぼ無し(y クランプ)。
- 背景 `bg_sky` は `tileSprite` + `setScrollFactor(0)` で固定し、`tilePositionX` をカメラ scrollX の
  0.3 倍で動かして簡易視差を付与。

### 自前物理(重力 + AABB + 反射)

- **物理エンジンは使わない**。各エンティティは中心座標 (x,y) と速度 (vx,vy) を持ち、
  毎フレーム `delta` 秒で更新する純粋な自前ループ。
- **軸分離 AABB**: `moveBody()` で X 移動 →衝突解決、Y 移動 →衝突解決の順に処理。
  衝突時はタイル境界へスナップし、下方向衝突を「接地」、上方向衝突を「天井ヒット」として返す。
  接地フラグで「ジャンプは接地時のみ」を実現。
- **可変ジャンプ**: 上昇中(vy<0)にジャンプキーを離すと vy を係数 0.45 で減衰させ、押し続けると高く跳ぶ。
- **敵**: 重力で落下し足場に着地。壁に当たる(`hitX`)と反転。さらに進行方向の足元タイルが空なら
  反転する簡易ガケ落下回避を実装。穴へ落ちた敵は非アクティブ化。
- **踏みつけ / 被弾**: プレイヤー AABB と敵 AABB の重なりを判定し、落下中(vy>0)かつ上面付近なら踏みつけ
  (撃破 +100・跳ね返り vy=-380・スパーク)、それ以外は被弾(HP-1・ノックバック・無敵 1.0s)。
- **穴落下 / 復帰**: 自機が `WORLD_H + 余白` を越えたら HP-1 してスポーン復帰。HP0 でも HP を 3 に戻して
  復帰し、スコアと敵はベンチ継続のため保持する。

### 決定的生成(シード固定)

- マップ・コイン・敵スポーンはすべて `mulberry32` 製の **決定的疑似乱数**で生成し、`Math.random` は不使用。
- マップ(seed `0x5A1701`): 最下段 2 行を地面とし、決定的な位置に穴(gap)を開ける。空中にブロック足場、
  地上に高さ 2〜3 の土管、左右端は壁柱。生成時に「敵/コインを乗せられる水平足場」リストも構築。
- コイン(seed `0xC014`): 各足場の上空 1 タイルに確率配置。
- 敵(seed `0xE9E33`): 足場リストから決定的に選んで配置。`+`/`-` での増減は index を seed に混ぜた
  PRNG で安定した位置を得るため、増減しても配置が決定的に再現される。
- 敵は**オブジェクトプール**で管理(`active` フラグで使い回し)。`syncEnemies()` で設定値まで
  スポーン / 末尾からトリムし、最大 500 体までの負荷比較に対応。

### アセット / フォールバック

- `../assets/`(= `5/assets/`)から SPEC のファイル名で読込。`BootScene` で `loaderror` を捕捉し、
  欠落キーは `Graphics.generateTexture` で焼いた図形テクスチャにフォールバック
  (自機=赤矩形 / goomba=茶丸 / 地面=茶 / ブロック=橙 / 土管=緑 / コイン=黄丸 / 背景=空色)。
  **画像が 1 枚も無くても必ず起動する。**

## Codex 生成所感(自前物理プラットフォーマーの実装しやすさ)

- **自前物理 vs Arcade**: Phaser には Arcade Physics があるが、本サンプルは「重力・AABB・反射を可視化して
  比較する」目的のため意図的に物理エンジンを使わず自前実装した。軸分離 AABB(X→解決 / Y→解決)は
  プラットフォーマーの定番パターンで、接地フラグと境界スナップさえ丁寧に書けば Phaser 側の API には
  ほぼ依存せず素直に書ける。Arcade を使えば重力・衝突は数行で済むが、当たり挙動(踏みつけ判定・
  可変ジャンプ・ガケ回避)を細かく制御したい比較用途では自前のほうが見通しが良かった。
- **タイルカリング**: テーマ3と同じく `worldView` から可視範囲が 1 行で取れ、Blitter で
  「可視タイルだけ Bob を作る」カリングが完結する。横長 6400px マップでも常に数百タイルしか描かず、
  `Tiles drawn` として描画数をそのまま HUD に出せるのが性能比較と好相性。
- **カメラ追従**: `setBounds` + `startFollow` の 2 行で水平追従とスクロール範囲クランプが完結。
  横スクロールアクションの土台作りは短時間で済んだ。
- **多数エンティティ**: 敵はオブジェクトプール + 自前 AABB ループで、500 体規模でも素直にスケールする。
  Container にまとめて深度管理し、`active` フラグで使い回す構造は Codex 生成でも安定して書けた。
- **フォールバック**: `Graphics.generateTexture` が標準であるため、アセット欠落時の図形フォールバックを
  他エンジンより手軽に焼ける点は引き続き Phaser の強み。
- 総じて Phaser 4 は「広い横長マップ + カメラ + 自前カリング + 自前物理」をシンプルに両立でき、
  API 名(blitter / worldView / generateTexture / startFollow)も安定していて迷いの少ない実装になった。
