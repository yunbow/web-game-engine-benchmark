# テーマ5 横スクロールアクション ― LittleJS 版

横スクロールジャンプアクションの横スクロールジャンプアクションを **LittleJS** で実装したもの。
200x17 タイル（6400x544px）の決定的マップを可視範囲のみカリング描画し、
**重力・AABB タイル衝突・カメラ水平追従・多数の敵更新**を自前実装して、
「広い横長タイルマップ描画 ＋ 重力/AABB物理 ＋ 多数エンティティ」の同時負荷を測る。

## 起動方法

CDN 読込のため `file://` で直接開いても動くが、`../assets/` の相対参照と
ブラウザのキャッシュ挙動を安定させるため簡易HTTPサーバ経由を推奨。

```bash
# このフォルダ (5/LittleJS) で
python -m http.server 8000
# → ブラウザで http://localhost:8000/ を開く
```

`../assets/` フォルダは空でも起動する（画像欠落時は単色図形フォールバック）。
SPEC のアセット（`player.png` 等）を `../assets/` に置けば自動で使用される。

## 使用バージョン / CDN

- **LittleJS**（`@latest` 解決時点 / classic global build）
- 主URL: `https://unpkg.com/littlejsengine`
  失敗時フォールバック: `https://cdn.jsdelivr.net/npm/littlejsengine/dist/littlejs.min.js`
- 素の unpkg は classic global build（グローバル `engineInit` 等を生やす版）を解決するため
  `<script>` 直読みできる。読込失敗時のみ `error` イベントを捕捉して jsDelivr ミラーへ切替。

## 操作

- **← → / A D**: 水平移動（歩き 180px/s）。
- **Space / ↑ / W**: ジャンプ（接地時のみ・**可変ジャンプ**: 上昇中に離すと上昇減衰）。
- **Shift**: ダッシュ（×1.6 = 288px/s）。
- **+ / -**（メイン行 or テンキー）: ワールド上の敵数を 10 ずつ増減（負荷調整・性能比較の主軸）。

## HUD（常時表示・左上, HTML `#hud` overlay）

- `FPS`: 実測の移動平均（エンジン `frameRate` を指数移動平均で平滑化）。
- `Tiles drawn`: そのフレームで実際に描画したタイル数（カリング後）。
- `Entities`: 敵 + 未取得コインの合計。
- `Player (x, y)`: 自機のタイル座標。
- `Score` / `Coins` / `HP`。
- `Enemies`: 現在の敵数 / 設定値。
- `[sprites]` / `[shapes fallback]`: 画像使用かフォールバックかを明示。

## 実装メモ

### Y軸 / 座標系
LittleJS は **Y軸が上向き**。テーマ3 LittleJS と同じ思想で、`cameraScale=1`
（1ワールド単位 = 1px）とし、ゲーム論理も「px・Y上向き」の一貫モデルで保持する
（画面下 = Y小, 画面上 = Y大）。タイル配列 row 0 を画面上に出すため
`worldY = WORLD_H - ty*TILE` で上下反転し、変換は `tileCenterWorld()` 等に閉じ込めた。

- **重力**は画面下（=Y減少）方向なので速度に `-GRAVITY` を積む。
- **ジャンプ初速**は SPEC では -640（上）だが、本実装の y-up では上昇 = +Y のため `+640`
  として `JUMP_VEL` 定数に符号規約を一本化。入力・物理・カメラ・座標表示すべてを
  同じ規約で通し、「上を押すと下に行く」「片側だけカリングが欠ける」事故を防いだ。
- **カメラ**は `setCameraPos` で自機を水平追従（x を `[VIEW_W/2, WORLD_W-VIEW_W/2]` でクランプ）。
  高さ 544 ≈ 画面 540 のため y は `WORLD_H/2` 固定でほぼ縦スクロール無し。

### カリング
`gameRender` でカメラ中心 ± 画面半分（+余白1タイル）から **タイルX範囲**を算出し、
その範囲だけ二重ループで `drawTile`/`drawRect`。縦はマップ全高（17 タイル）が
画面内に収まるため全行描画。実描画数を `Tiles drawn` に集計（≈ 30x17 ≒ 500 枚）。
敵・コインも画面外を `continue` でスキップ。

### 自前物理（重力 + 軸分離 AABB）
エンジン物理（`setGravity` ＝ 0 で無効化）は使わず自前実装。
- `movePlayer`: **X 移動 → 解決, Y 移動 → 解決**の軸分離。solid タイルと重なったら
  進行方向のタイル境界へスナップ（壁ずり / 着地 / 天井）。着地時に `grounded=true`。
- ジャンプは `grounded` 時のみ。可変ジャンプは上昇中（vy>0）にキーを離すと `vy *= 0.45`。
- 敵 `moveEnemy` も同じ軸分離 AABB。X で壁に当たると `vx` 反転、Y は重力で落下し着地。
- `boxHitsSolid` が AABB の覆うタイル矩形を走査して solid 有無を返す共通判定。

### 決定的生成
`mulberry32` の決定的乱数（マップ固定シード `20260614`, コイン/敵も別シード固定）。
最下段 2 行を地面にしつつ散発的・周期的な穴（gap）、中段に空中ブロック足場、
地上に高さ 2〜3 の土管、左右端は地面で壁。`Math.random` は不使用。
敵スポーンは「地面のある列の上面 1 タイル上」を候補列として事前生成し、
`+`/`-` では先頭から `enemyTarget` 体を採用するので**増減しても配置が決定的**。

### テクスチャ判定 / フォールバック
`textureInfos[i].size.x > 1` で読込済みを判定（`spriteReady`）。読込済みなら
`tile(0, size, i)` で `drawTile`、未読込なら `drawRect`/`drawCircle`
（自機=赤 / goomba=茶丸 / 地面=茶 / ブロック=橙 / 土管=緑 / コイン=黄丸 / 背景=空色）。
画像が 1 枚も無くても必ず起動する。

## Codex 生成所感（横スクロール物理を軽量エンジンで書く所感・罠）

- **Y軸上向きと「地形物理」の相性が一番の論点**。テーマ3はマップを眺めるだけなので
  Y反転を描画に閉じ込めれば済んだが、本テーマは重力・着地・天井・踏みつけが絡む。
  「下降 = vy 減少」「着地は床の*上面*へスナップ」「天井は床の*下面*へ」を y-up のまま
  一貫させる必要があり、符号を一箇所でも混ぜると自機が床にめり込む/天井を突き抜ける。
  `JUMP_VEL` と `GRAVITY` の符号規約を定数に固定し、スナップ計算を `WORLD_H - row*TILE`
  という単一の式系で統一したのが効いた。
- **軸分離 AABB は LittleJS には用意が無い**ので完全自前。逆に抽象が薄いぶん、
  「X を動かして解決 → Y を動かして解決」と素直に書け、接地判定も Y 解決の副産物として
  自然に取れる。Phaser の Arcade Physics のような自動解決が無い代わりに、
  めり込み量・スナップ先が完全に見えるのは性能比較用途では利点。
- **カリングは横方向だけで済む**のが横スクロールの良いところ。縦 17 タイルは画面に
  収まるので全行描画固定にでき、`Tiles drawn` がほぼ一定（≒ 可視幅×17）で
  「描画コストの下限」を観測しやすい。負荷は専ら `+/-` の敵数で振れる。
- **決定的な敵配置の維持が地味に重要**。`+/-` で都度ランダム生成すると比較にならないため、
  候補列を一度だけ生成し「先頭 N 体」を採る方式にした。これで 20→500 体まで
  同じ並びのまま体数だけ増え、FPS の素直な比較ができる。
- **HUD は HTML overlay が最も素直**。テーマ3は WebGL を切って `mainContext` 直描きしたが、
  本実装はテーマ1同様 `#hud` div へ `textContent` を流すだけにした。WebGL レイヤとの
  重なりを気にせず、行数の多い HUD（FPS/Tiles/Entities/座標/Score/Coins/HP/Enemies）も
  `white-space: pre` で素直に整形できる。
- 総じて LittleJS は「薄い API を正しく組む」エンジン。物理が無いぶん横スクロールの
  当たり判定を自分で完全制御でき、性能の所在（タイル描画 vs 敵更新）が HUD にそのまま
  出るのが、ベンチ題材として扱いやすかった。
