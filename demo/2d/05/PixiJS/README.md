# テーマ5 横スクロールアクション ― PixiJS v8 版

横スクロールのジャンプアクション（横スクロールジャンプアクション）を **PixiJS v8** で実装。
200×17 タイル（6400×544px）の横長マップを可視範囲のみカリング描画し、重力／AABB物理／
多数の敵更新を同時に回す性能比較用デモ。

## 起動方法

ローカルにHTTPサーバを立ててブラウザで開く（`PIXI.Assets.load` が `file://` では CORS で失敗するため）。

```bash
# このフォルダ(5/PixiJS/)の1つ上(5/)で実行すると ../assets/ も解決できる
cd c:/work/claude/local-works/web-game/5
python -m http.server 8000
# → http://localhost:8000/PixiJS/ を開く
```

`../assets/` に画像が無くても **Graphics 図形フォールバック** で必ず起動する。

## 使用バージョン

- PixiJS **v8**（CDN: `https://cdn.jsdelivr.net/npm/pixi.js@8/dist/pixi.min.js`）
- 追加ライブラリなし。`index.html` + `game.js` の2ファイル構成。
- **v8 注意**: 初期化は `const app = new PIXI.Application(); await app.init({...});` の2段階。
  キャンバスは `app.view` ではなく **`app.canvas`**。Graphics は新チェーンAPI
  （`gr.rect(...).fill(color)` / `.stroke({width,color})`）。フォールバックは
  `app.renderer.generateTexture({ target: graphics })` でテクスチャ化して Sprite 再利用。
- **物理エンジンは未使用**。重力・AABB・反射はすべて自前。

## 操作

| 入力 | 動作 |
|---|---|
| `←` / `→`、`A` / `D` | 移動（歩き 180 px/s） |
| `Space` / `↑` / `W` | ジャンプ（接地時のみ・可変ジャンプ） |
| `Shift` | ダッシュ（288 px/s = ×1.6） |
| `+` / `-`（テンキー可） | 敵数を ±10 増減（初期20体・下限0・上限500） |

## 仕様数値（全エンジン共通）

- キャンバス **960×540**、タイル **32×32**、マップ **200×17**（6400×544px）。
- マップは固定シード（`mulberry32(20250614)`）で**決定的生成**。
  `0=空 / 1=地面 / 2=ブロック / 3=土管`、進入不可（solid）= 1,2,3。
- 重力 **1800 px/s²**、ジャンプ初速 **-640 px/s**（上昇中にキーを離すと×0.45で減衰）。
- 自機: 当たり判定 **24×44**（描画 32×48）、歩き **180** / ダッシュ **288 px/s**、HP **3**。
- 敵（goomba）: 当たり判定 **28×28**、水平 **60 px/s**、重力あり、壁で反転。
  - 踏みつけ（自機 vy>0 で上面着地）→ 撃破 **+100**・火花・自機跳ね返り **vy=-380**。
  - 横接触 → 被弾（ノックバック＋無敵 **1.0s**・**HP-1**）。
- コイン: 決定的配置、接触取得で消える（**+50** / `Coins++`）。`Entities` に含む。
- 穴落下（y > マップ下端 + 余白80px）または HP0 → スポーン地点へ復帰（HP3に戻す。スコア・敵は保持）。

## HUD（HTMLオーバーレイ `#hud`・約120msごと更新）

- `FPS`（直近60フレームの移動平均）
- `Tiles drawn`（そのフレームで実描画したタイル数）/ `Entities`（敵 + 未取得コイン）
- 自機タイル座標 `(x, y)`
- `Score` / `Coins` / `HP`
- `Enemies`（現在数 / 設定値・上限500）

HUD はキャンバス外の HTML `<pre>` 要素で描画。Pixi の描画負荷と切り離して計測できる。

## 実装メモ

- **マップ生成**: 最下2段を「地面ラン（5〜10連続）」と「穴（幅2〜3）」を交互に決定的生成。
  空中にブロック足場（70箇所）、地上に土管（24箇所・地面上に2段）、左右端は全高の壁。
- **タイルカリング（本デモの核）**: 画面に収まる最大タイル数ぶん（`(960/32+2)×(540/32+2)`）の
  `Sprite` をプールし、毎フレーム可視範囲のタイルへテクスチャと座標を割り当てて**再利用**する。
  空タイルはスキップするため `Tiles drawn` は地形密度に応じ約 300〜500。`TilingSprite` を使わない真のカリング。
- **自前物理（重力＋AABB軸分離）**: アクター `{x,y,w,h,vx,vy,onGround}` に対し、
  X移動→solidタイルへスナップ解決 → Y移動→解決、の順に **軸を分離**して当たりを取る。
  Y方向の下衝突で `onGround=true` を立て、ジャンプは接地時のみ許可。可変ジャンプは
  上昇中（vy<0）にジャンプキーを離した瞬間に `vy*=0.45` で減衰させる素朴な実装。
- **敵プール**: goomba スプライトは破棄せず `enemyPool` に退避して再利用。`+/-` の増減は
  決定的にシャッフルした `spawnSlots`（直下が solid な空きセル）を順に使い、末尾から削除する。
  500体でも sprite 生成は初回のみ。
- **当たり判定**: 自機×敵は AABB。落下中かつ足が敵上面付近なら踏みつけ（撃破＋跳ね）、
  それ以外は横接触（被弾）。コインはオーバーラップで取得。
- **カメラ**: 自機中心の**水平追従**＋`[0, worldW-960]` クランプ。`world` コンテナを `-camX` 平行移動。
  高さ544≈画面540のため縦はほぼクランプ。背景は `TilingSprite` を `camX*0.4` でずらし軽い視差。
- **フォールバック**: 各アセットを `PIXI.Assets.load` で試行し、失敗時は `generateTexture` 生成の
  図形テクスチャに差し替え（自機=赤 / goomba=茶 / 地面=茶 / ブロック=橙 / 土管=緑 / コイン=黄 / 背景=空色）。

## Codex 生成所感（v8 で横スクロール・自前物理を組む観点）

- **v8 初期化の罠（再掲）**: `new PIXI.Application(opts)` は無視され `app.canvas` が生えない。
  必ず `new PIXI.Application()` → `await app.init(opts)` の2段階。`app.view` も `app.canvas` に改名。
- **カリングとの相性は◎**: テーマ3（トップダウン）と同じく「Sprite を並べてコンテナを `-cam` 平行移動」
  という素直なモデルなので、横長6400pxマップでも可視ぶん（≦`32×19`スプライト）に圧縮でき、
  実描画タイル数を `Tiles drawn` で完全制御できる。広いマップを軽快にスクロールできた。
- **自前物理は描画ライブラリ側に何も用意が無い**ぶん、軸分離AABBを丁寧に書く必要がある。
  ただし Pixi は当たり判定に一切干渉しないので、「論理座標（x,y,w,h）で物理 → 反映時に
  描画オフセット（24×44 判定を 32×48 描画の中央下へ）」と**論理と描画を完全分離**でき、
  仕様数値どおりの挙動を再現しやすかった。タイルスナップは `floor((x+w)/TILE)*TILE-w` の素朴解決で十分。
- **HUD を HTML オーバーレイにした利点**: Pixi の `Text` で出すとテキスト更新もGPU負荷に乗るが、
  HTML `<pre>` を被せれば描画計測（FPS / Tiles drawn）と HUD 更新が独立する。120ms間引きで十分滑らか。
- **多数エンティティの更新**: 敵は単純な重力＋反転だけなので、500体でも更新ループ自体は軽い。
  ボトルネックは概ねスプライトのワールド変換更新側に出る。プール再利用＋`visible=false` 退避で
  GC とドローコール増を抑えられ、`+/-` での負荷比較が安定して取れた。
- 総じて v8 は「描画は超軽量・物理は完全自前」という分担が明快で、横スクロールアクションの
  ベンチ題材としては**カメラ／カリングが書きやすい一方、物理は他エンジン（Phaser/Godot）より手数が多い**
  という比較軸がはっきり出る構成だった。
