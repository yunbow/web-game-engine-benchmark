# 横スクロールアクション — KAPLAY 版

テーマ5共通仕様（横スクロール・重力/AABB物理・広い横長マップのカリング・性能比較用）の **KAPLAY** 実装。
画像アセットが無くても単色図形フォールバックで必ず起動します。

使用バージョン: **KAPLAY 3001.0.19**（CDN: `unpkg.com/kaplay@3001.0.19/dist/kaplay.js`）。

## 画面・マップ
- キャンバス **960 x 540**（固定）/ タイル **32x32**。
- マップ **200 x 17 タイル**（= **6400 x 544 px**）を固定シードで決定的生成。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`5/`)をルートに HTTP サーバを立てて開く。

```bash
cd 5
python -m http.server 8000
# → http://localhost:8000/KAPLAY/
```
`file://` 直開きは画像が CORS で読めない（図形フォールバックでは起動する）。

## 操作
- 移動: `←`/`→` または `A`/`D`
- ジャンプ: `Space` / `↑` / `W`（接地時のみ・可変ジャンプ）
- ダッシュ: `Shift` 押下中（1.6倍速）
- `+` / `-`（テンキー +/- も可）: 敵数を ±10（下限0・上限500）

## 実装メモ
- `kaplay({ canvas, width:960, height:540, global:false })` で初期化。名前空間 `k.*` を明示利用。
- KAPLAY の座標系は **Y 下向き・原点左上 = 画面座標と一致**するため、PixiJS 正準実装と同じ画面座標でゲームロジックを書ける。
- **物理は SPEC 準拠の自前実装**: 重力 1800 / 可変ジャンプ / **AABB を軸分離（X→解決, Y→解決）** したタイル当たり。KAPLAY の `body()`/`area()` は使わず他エンジンと同条件にした。
- **広い横長マップの水平スクロール**: ワールド→画面の `camX,camY` オフセットを毎フレーム算出（自機中央, `camX∈[0, 6400-960]` でクランプ）。描画は `onDraw` 内で `k.pushTranslate(-camX,-camY)` してからワールド座標で即時描画する。
- **タイルカリング**: 可視範囲（`ceil(960/32)+2` 列 × `ceil(540/32)+2` 行 ≈ 32x19）のみを毎フレーム走査し、空気以外のタイルだけ `drawSprite`/`drawRect` で描画（描画数を `Tiles drawn` として HUD 表示）。敵・コインも可視 x 範囲のみ描く。
- アセットは `loadSprite` を個別 try/catch し、失敗したものだけ図形（`drawRect`/`drawCircle` + 色）にフォールバック。
- HUD は HTML オーバーレイ（`#hud`）に **FPS / Tiles drawn / Entities / 自機タイル座標 / Score / Coins / HP / Enemies(現在/設定)** を表示。

## Codex / AI コーディング所感
- ミニマル API で「マップ生成〜ループ〜入力〜描画」を一気通貫で生成しやすい。
- 罠: ① 旧 **Kaboom** との API 差で記憶が混ざる。② キー名が独自（`"left"`, `"shift"`, `"kpadd"`, `"minus"`）で `e.code` ではない。③ 広いマップでは **カメラ/ワールドオフセットを一貫させる**必要があり、`k.add` で大量タイル実体を作ると破綻するため、本実装は `onDraw` の即時描画 + プールなしカリングにした。④ `body()`/`area()` を使うと当たり判定方式が他エンジンと変わるため、軸分離 AABB を自前実装する点は明示指示が要る。
