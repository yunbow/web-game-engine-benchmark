# テーマ5 共通仕様 ― 横スクロールアクション

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 横スクロールのジャンプアクション（横スクロールジャンプアクション）。重力・タイル地形との当たり判定・カメラ追従を持つ。
- 敵（クリボー風）を踏んで倒し、コインを集めながらゴール方向へ進む。
- **目的**: 性能比較。**広い横長タイルマップ描画（カリング）＋ 重力/AABB物理 ＋ 多数の敵の更新**を同時に回した性能を測る
  （テーマ3の「大マップ描画」に重力物理と多数エンティティを足した題材）。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。タイルサイズ **32x32**。
- マップ: **200 x 17 タイル**（= 6400 x 544 px）。配列で定義。決定的に生成（固定シード）。
  - `0=空(空気)`, `1=地面(ground)`, `2=ブロック(brick)`, `3=土管(pipe)`。
  - **進入不可（solid）= 1, 2, 3**。`0` は通過可。
  - 最下段は地面、所々に**穴（gap）**、空中に**ブロック足場**、地上に**土管**を配置。左右端は壁。
- カメラは自機を**水平追従**（自機が概ね中央, x を `[0, worldW - 960]` でクランプ）。
  高さ 544 ≈ 画面 540 のため縦スクロールはほぼ無し（y はクランプ）。
- **可視範囲のタイルのみ描画**（カリング）すること（約 30x17 ≈ 510 タイル）。

## 操作
- 移動: 矢印 `←`/`→` または `A`/`D`。
- ジャンプ: `Space` / `↑` / `W`（接地時のみ）。
- ダッシュ: `Shift` 押下中は走り（1.6倍速）。
- `+` / `-`: **画面/ワールド上の敵数（負荷）を増減**。性能比較の主軸。

## ルール / 数値（全エンジン共通）
- 物理: 重力 **1800 px/s²**。デルタタイム基準。
- 自機: 当たり判定 **24w × 44h**（描画 32x48）。
  - 歩き速度 **180 px/s**、ダッシュ **288 px/s**（×1.6）。
  - ジャンプ初速 **-640 px/s**（上）。**可変ジャンプ**（上昇中にジャンプキーを離すと上昇を減衰）。
  - 地形との当たりは **AABB を軸分離**（x移動→解決, y移動→解決）。接地フラグを持つ。
- 敵（goomba 風）: 当たり判定 **28x28**、水平歩行 **60 px/s**、重力あり。
  **壁／足場の端で反転**（簡易: 壁衝突で反転。任意でガケ落下回避）。
  - **踏みつけ**: 自機が落下中（vy>0）に敵の上面に乗ると敵撃破、自機は跳ねる（vy=**-380**）。**Score +100**。撃破時に `hit_spark`。
  - **横接触**: 自機が被弾（ノックバック＋無敵1.0s, **HP-1**）。
- コイン: マップ上に決定的に配置。接触で取得 → `Coins++`, **Score +50**（取得で消える）。`Entities` に含める。
- 自機 HP 初期 **3**。**穴に落下**（y > マップ下端 + 余白）または HP0 で**スポーン地点へ復帰**
  （HP を3に戻す。ベンチ継続のためスコア・敵は保持）。
- 物理エンジンは使わず、重力・AABB・反射を自前実装する。

## 敵数（負荷）
- 初期 **20** 体、`+`/`-` で **±10**（下限0・上限 **500**）。
  足場のある地形にスポーンし、決定的な配置から増減する。

- `Enemies`（現在の敵数 / 設定値）
