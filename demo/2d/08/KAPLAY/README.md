# パーティクル / 魔法エフェクトdemo — KAPLAY 版

テーマ8共通仕様（パーティクル/魔法エフェクトデモ・性能比較用）の **KAPLAY** 実装。
画像アセットが無くても放射状グロー（canvas）フォールバックで必ず起動します。

使用バージョン: **KAPLAY 3001.0.19**（CDN: `unpkg.com/kaplay@3001.0.19/dist/kaplay.js`）。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`8/`)をルートに HTTP サーバを立てて開く。

```bash
cd 8
python -m http.server 8000
# → http://localhost:8000/KAPLAY/
```
`file://` 直開きは画像が CORS で読めない（グローフォールバックでは起動する）。

## 操作
- マウス移動: カーソル追従トレイル（連続噴出）
- 左クリック: その地点で爆発バースト（120〜200個を放射状に放出）
- `Space`: オート花火トグル（ON 中・0.5s 間隔・決定的位置にバースト）
- `B`: ブレンド切替（加算 ⇄ 通常）
- `+` / `-`（テンキー +/- も可）: 目標同時パーティクル数 ±2000（500〜50000）
- `R`: リセット

## 使用したパーティクル機構（重要）
- **CPU パーティクル（自前配列 + `onDraw` 手描き）**。
  - KAPLAY には **GPU パーティクルシステムが無い**。`add([...])` でゲームオブジェクト化すると
    1 個 = 1 オブジェクトとなりスケールしない。そこで **LittleJS と同様に自前のパーティクル配列
    （SoA 風の軽量オブジェクト + free スタック）を JS で更新**し、毎フレーム `onDraw` で
    `drawSprite` を 1 個ずつ呼んでまとめて描画する **CPU 方式**を採用。
  - これは three.js / A-Frame の **GPU（THREE.Points / 単一 draw call）方式と対比**する
    「CPU 更新 + パーティクル数ぶんの描画コール」のサンプル。**本デモの比較軸そのもの**であり、
    CPU 方式のため **GPU エンジンより低い目標数で FPS が頭打ちになる**（＝正当な比較結果）。
- **加算ブレンド**: `drawSprite({ blend: k.BlendMode.Add })` を指定（KAPLAY 3001 系）。
  - 白基調の放射グローテクスチャを暗背景に重ね、重なるほど明るく光る。`B` で通常合成へ切替
    （`blend` 未指定）。`BlendMode.Add` が無い環境では通常合成にフォールバックするが、
    白グロー × 暗背景なので近い見た目にはなる。
- 寿命に沿って **size: 大(1.4)→小(0.15)**、**alpha: 1→0（終盤 t² で急減）**、**色: 暖色→寒色**を
  CPU 補間。物理は軽い重力 + 減速。プール再利用で GC を避ける。

## 実装メモ
- `kaplay({ canvas, width:960, height:540, background:[8,8,15], global:false })` で初期化。名前空間 `k.*`。
- 座標系は **Y 下向き・原点左上 = 画面座標と一致**（three/A-Frame と違い変換不要で最も素直）。
- 決定的擬似乱数 `mulberry32`（`Math.random` 不使用）で周回オーブ軌道・バースト方向を再現可能に。
- 周回オーブ4個は固定シードの正弦合成で決定的に周回し、火花を連続噴出（マウス無しでもベンチ安定）。
- HUD は HTML オーバーレイ（`#hud`）に FPS / Particles(live) / Target / Emitters / Blend / Mode を表示。

## Codex / AI コーディング所感
- ミニマル API で「初期化〜ループ〜入力〜描画」を一発生成しやすい。
- 罠: ① KAPLAY は **GPU パーティクルが無い**ため、テーマ8の「GPU パーティクル比較」では
  CPU 方式を選ぶしかない点を明示しないと、AI が存在しない GPU API を捏造しがち。
  ② `drawSprite` のブレンド指定方法（`blend` プロパティ / `BlendMode`）はバージョン差があり、
  存在ガードを入れないと壊れる。③ 旧 Kaboom との API 混同。
