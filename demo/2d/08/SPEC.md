# テーマ8 共通仕様 ― パーティクル / 魔法エフェクトデモ（GPUパーティクル × ブレンド × post）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 多数のエミッタが加算合成のパーティクル（火花・魔法）を噴き出すエフェクトデモ。マウスで魔法弾を撃ち、着弾で爆発バーストが出る。
- **目的**: 性能比較。**各エンジンが推奨するパーティクルシステム（できれば GPU）と加算ブレンドの描画スループット**を測る。
  - テーマ1の「スプライト枚数」とは別軸の、**加算合成（additive blend）・寿命に沿う色/サイズ変化・大量パーティクル**を比較する。
- 性能比較の核は「**画面上の生存パーティクル総数**」。

## 使用するパーティクル機構（エンジンごとに“素直な選択”を使う ＝ これ自体が比較対象）
- **Phaser4**: `GameObjects.Particles.ParticleEmitter`（内蔵パーティクル）。
- **PixiJS**: `ParticleContainer` ＋ 加算ブレンド（`BLEND_MODES.ADD`）でスプライトを大量描画（自前更新）。
- **Babylon.js**: `ParticleSystem` / 可能なら `GPUParticleSystem`（GPU対応時は GPU を優先）。
- **LittleJS**: 内蔵の `ParticleEmitter`（加算ブレンド対応）。
- **Godot**: `GPUParticles2D`（不可環境向けに `CPUParticles2D` フォールバックを用意してよい）。
- いずれも**加算合成**で発光感を出す。寿命に沿って **alpha フェードアウト＋サイズ変化＋色グラデ**を行う。
- 使用機構・GPU/CPU の別・つまずきを各 README に必ず記載する。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。背景は暗色（`#08080f` 相当）で発光が映えるようにする。
- 時間はデルタタイム基準。
- **常設エミッタ**: 画面内を**決定的な軌道で周回する数個（初期4個）の発光オーブ**から、火花パーティクルを連続噴出。
  - オーブの軌道は固定シードの正弦/円運動など決定的に定義（マウス無しでも常にパーティクルが流れる＝ベンチ安定）。
- **背景**: 微弱に明滅する星/グリッド等を任意で薄く描いてよい（負荷の主役はパーティクル）。

## 操作
- **マウス移動**: カーソル位置に小さな追従エミッタ（トレイル）を出す。
- **左クリック**: クリック地点で**爆発バースト**（一度に多数のパーティクルを放射状に放出）。
- `Space`: **オート花火**トグル。ON の間、**0.5s 間隔で決定的な位置に爆発バースト**を自動生成（マウス無しでベンチ可）。
- `B`: ブレンドモード切替（**加算 ⇄ 通常**）。描画コスト/見た目の比較用（任意機能だが推奨）。
- `+` / `-`: **エミッション量（1秒あたりの発生レート ＝ 目標パーティクル数）を増減**。性能比較の主軸。
- `R`: リセット。

## ルール / 数値（全エンジン共通）
- 各パーティクル: 寿命 **0.6〜1.4s**、初速はランダム方向（決定的PRNG）、重力 or 減速を軽く適用、
  寿命に沿って **size: 大→小、alpha: 1→0**、色は暖色〜寒色のグラデ。**加算合成**で重なるほど明るく。
- 常設の周回オーブは **連続レート噴出**、クリック/オート花火は **一度に 120〜200 個の放射バースト**。
- パーティクルは**プール再利用**（生成/破棄のGCを避ける）。生存上限に達したら最古を再利用。
- これはゲームではなくデモのため、スコアやライフは無し（代わりにパーティクル統計をHUDに出す）。

## パーティクル数（負荷）
- 目標同時パーティクル数の上限 初期 **2000**、`+`/`-` で **±2000**（下限 **500**・上限 **50000**）。
  - エミッション量（レート）はこの上限を満たすように調整。`Particles` 実測値が上限付近で安定する。
  - 増やすほど更新（位置/alpha/size）と加算ブレンド描画のスループットが効く。GPUパーティクル対応エンジンと
    CPU更新エンジンで `FPS` の落ち方に差が出る。

## アセット（`../assets/` から読み込み）
| ファイル | 用途 | 推奨px |
|---|---|---|
| `particle_spark.png` | 火花（加算合成前提・中心白の放射状グロー） | 32x32 |
| `particle_smoke.png` | 煙/もや（任意の第2テクスチャ） | 32x32 |
| `orb.png` | 周回オーブ（エミッタ本体） | 32x32 |
| `bg_dark.png` | 背景（暗色, タイル可） | 512x512 |

- **画像が無い場合は単色図形でフォールバック**（パーティクル=放射グラデの白丸 / オーブ=発光円 / 背景=暗色）。
  - 図形フォールバック時も**加算合成**は維持する。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Particles`（現在生存しているパーティクル数）/ `Target`（目標上限）
- `Emitters`（常設オーブ数 + アクティブバースト数）
- `Blend`（ADD / NORMAL）/ `Mode`（GPU / CPU いずれを使ったか）
- 画面下に操作ヒント行（クリック=爆発 / Space=オート / B=ブレンド / +/-=量 / R=リセット）。
