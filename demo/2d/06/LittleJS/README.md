# テーマ6 タワーディフェンス(経路探索) ― LittleJS 版

グリッド上のタワーディフェンスを **LittleJS** で実装したもの。
30x17 タイル(960x544px)の決定的マップ上で、敵がスタート(左中央)から
**A\* 経路探索**でゴール(右中央)を目指し、タワーが射程内の敵を撃つ。
タワー設置/撤去のたびに **A\* を再計算し、全 creep が一斉に再追従**するため、
「グリッド経路探索 ＋ 多数ユニットの追従更新 ＋ タワー射撃」の同時負荷を測る。

## 起動方法

CDN 読込のため `file://` で直接開いても動くが、`../assets/` の相対参照と
ブラウザのキャッシュ挙動を安定させるため簡易HTTPサーバ経由を推奨。

```bash
# このフォルダ (6/LittleJS) で
python -m http.server 8000
# → ブラウザで http://localhost:8000/ を開く
```

`../assets/` フォルダは空でも起動する(画像欠落時は単色図形フォールバック)。
SPEC のアセット(`creep.png` 等)を `../assets/` に置けば自動で使用される。

## 使用バージョン / CDN

- **LittleJS**(`@latest` 解決時点 / classic global build)
- 主URL: `https://unpkg.com/littlejsengine`
  失敗時フォールバック: `https://cdn.jsdelivr.net/npm/littlejsengine/dist/littlejs.min.js`
- 素の unpkg は classic global build(グローバル `engineInit` 等を生やす版)を解決するため
  `<script>` 直読みできる。読込失敗時のみ `error` イベントを捕捉して jsDelivr ミラーへ切替。

## 操作

- **左クリック**: マウス下の通路タイルにタワーを設置(25 Gold)。
  経路を完全に塞ぐ配置・start/goal・既設タイル・ゴールド不足は設置失敗。
- **右クリック**: タワーを撤去(返金なし・経路を再計算)。
- **+ / -**(メイン行 or テンキー): 同時出現する敵数の上限を 10 ずつ増減(負荷調整・比較の主軸)。
- **R**: リスタート(マップ・ゴールド・ライフ初期化)。

カーソル下のタイルは設置可=緑 / 不可=赤 でハイライトされる。

## HUD(常時表示・左上, HTML `#hud` overlay)

- `FPS`: 実測の移動平均(エンジン `frameRate` を指数移動平均で平滑化)。
- `Enemies`: 生存数 / 上限。
- `Towers`: 設置数 / `Projectiles`: 生存弾数。
- `Path recalcs`: 経路再計算の累計回数 / `Path len`: 現在経路のタイル長。
- `Gold` / `Lives` / `Score`。
- `[sprites]` / `[shapes fallback]`: 画像使用かフォールバックかを明示。

## 実装メモ

### Y軸 / 座標系(最重要)
LittleJS は **Y軸が上向き**。本題材は単一画面のグリッドで重力が無いため、
テーマ5 のような px・y-up モデルではなく、**「論理/グリッドは y-down(row 0 が画面上)」**
という素直なモデルで保持し、**描画と入力変換の境界でのみ Y を反転**する方針にした。

- `cameraScale = 1`(1ワールド単位 = 1px)+ `setCameraPos(WORLD_W/2, WORLD_H/2)` で
  960x544 グリッドを画面にそのまま収める(VIEW_H=540 との 4px 差は許容)。
- タイル(tx,ty)→ワールド中心: `worldY = WORLD_H - (ty*TILE + TILE/2)`(`tileCenterWorld`)。
  creep / projectile も論理は y-down の px で保持し、描画時に `scrToWorld()` で反転。
- **最大の罠は `mousePos` の変換**。`mousePos` は LittleJS ワールド座標(y-up)で来るため、
  タイルに直すには **先に `scrY = WORLD_H - mousePos.y` で y-down へ反転してから**
  `floor(scrY/TILE)` する(`mouseToTile`)。ここを反転し忘れると上下逆のタイルに
  タワーが建つ(theme-5 の y-up 罠と同根)。変換は `tileCenterWorld`/`scrToWorld`/
  `mouseToTile` の 3 関数に閉じ込め、ロジック側は一貫して y-down で書いた。

### A\* 経路探索
4方向・コスト1・マンハッタン距離ヒューリスティックの A\* を、バイナリヒープ(配列)で実装。
**壁 AND タワータイルの両方を通行不可**(`isWalkable`)として扱う。start/goal セル自体は
始点/終点として探索に含める。設置/撤去のたびに再計算し `Path recalcs` を++、
さらに全 creep を現在地から再追従させる(`recomputePathAndReroute`)。

- **設置拒否**: 左クリック時は仮にタワーを立てて `aStarSearch` を試し、経路が
  得られなければ配置を取り消す(ゴール到達不能になる配置を禁止)。
- マップ生成直後にも経路の存在を保証(無ければ中央ラインを掘る)。

### 多数ユニットの経路追従
creep は経路(タイル中心の列)を線形に辿る。各 creep は次に目指す path index(`seg`)を
持ち、1 フレームで複数ウェイポイントを跨げるようループで移動を消化する。
**経路が変わったら**、各 creep は経路上で現在地に最も近いセルを探し、その先を
次ウェイポイントに付け替える(`rerouteCreep`)。タワー狙いは射程内で
**進行度(`creepProgress` = 累積距離)が最大 = ゴールに最も近い**敵を選ぶ。

### プール / 決定的生成
creep・projectile は固定プール(`creepPool`/`projPool`)から貸出/返却して GC を抑制。
マップは `mulberry32`(固定シード `20260615`)で決定的に生成:外周を壁で囲み、
左右中央に開口、内部に縦の"櫛"壁(毎列どこか 1 マスを必ず空ける)+ 散発的な単発ブロック。
`Math.random` は不使用。スポーン位置・初期ステータスも決定的。

### テクスチャ判定 / フォールバック
`textureInfos[i].size.x > 1` で読込済みを判定(`spriteReady`)。読込済みなら
`tile(0, size, i)` で `drawTile`、未読込なら図形フォールバック
(敵=赤丸 / タワー=青矩形 / 弾=黄丸 / 通路=濃灰 / 壁=灰 / ゴール=緑 / spark=白丸)。
経路ハイライトは画像不要で薄い細矩形を重ねて常時可視化。画像が 1 枚も無くても必ず起動する。

## Codex 生成所感(経路探索 × 多数追従を軽量エンジンで書く所感・罠)

- **y-up とグリッド入力の相性が最大の論点**。テーマ5 は重力物理のため px・y-up を貫いたが、
  本題材は「マウスのワールド座標 → タイル」変換が肝で、`mousePos.y` を反転し忘れると
  上下逆にタワーが建つ。論理を y-down に寄せ、反転を 3 つの変換関数に閉じ込めることで
  「クリック位置とハイライトと A\* の通行判定」が常に一致するようにした。
- **A\* 再計算コストが負荷の主軸**。タワー設置/撤去のたびに 1 回 A\* を回し、さらに全 creep が
  `rerouteCreep` で経路を貼り直す。`+/-` で敵上限を上げるほど、再追従ループ ＋ 射撃対象
  探索(各タワー × 各敵)が効いてくるので、HUD の `Path recalcs` と `Enemies` が
  性能の所在をそのまま示す。ヒープ A\* にしておくと 30x17 程度では再計算がほぼ無視できる。
- **設置の事前検証(経路が残るか)** は「仮置き → A\* → ダメなら取消」で素直に書ける。
  LittleJS は薄い API なので、こうしたゲームロジックは完全自前だが見通しよく組める。
- **HUD は HTML overlay が最も素直**。WebGL レイヤとの重なりを気にせず、9 行の統計を
  `white-space: pre` で整形できる。GAME OVER だけは `drawTextScreen` でキャンバスに重ねた。
- 総じて LittleJS は「経路探索のような CPU 側 AI ロジック」を載せても、描画 API が薄いぶん
  ボトルネックが純粋にロジック側に出る。ベンチ題材として性能の切り分けがしやすかった。
