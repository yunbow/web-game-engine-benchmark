# タワーディフェンス（経路探索）— three.js 版

テーマ6共通仕様（タワーディフェンス：A\* 経路探索 × 多数ユニット追従・性能比較用）の **three.js** 実装。
画像アセットが無くても単色図形（canvas）フォールバックで必ず起動します。

使用バージョン: **three.js r184**（CDN: `unpkg.com/three@0.184.0/build/three.module.js`、importmap で `three` を解決）。レンダラは **WebGLRenderer**。エンジン以外の追加 CDN は無し。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`6/`)をルートに HTTP サーバを立てて開く。
**ESM（importmap + `<script type="module">`）なので `file://` 直開きは不可**（モジュール/CORS）。

```bash
cd 6
python -m http.server 8000
# → http://localhost:8000/three.js/
```

## 操作
- **左クリック**: 通路タイルにタワー設置（25 Gold）。経路を塞ぐ配置は不可。
- **右クリック**: タワー撤去（返金なし・経路再計算）。
- `+` / `-`（テンキー +/- も可）: 同時出現する敵数の上限を ±10（下限10・上限500）。**性能比較の主軸**。
- `R`: リスタート。

## 共通仕様の数値（全エンジン同一）
- キャンバス **960 x 540**、タイル **32x32**、グリッド **30 x 17**（決定的生成・固定シード `20250615`）。
- 敵: 半径10 / 70 px/s / HP30、スポーン間隔 0.5s。タワー: 射程96 / 連射0.6s / 弾10dmg / 弾速320。
- 初期 Gold 120 / Lives 20 / Score 0。撃破で Gold +5・Score +10。敵数上限 初期30、±10、下限10・上限500。

## 実装メモ（テーマ6の要点）
- **A\* は自前実装**（4方向・コスト1・マンハッタン距離）。three.js は描画ライブラリで経路探索を持たないが、本テーマの観測対象は「経路再計算回数」なので、全エンジンで同一の自前 A\* を共有する（組み込みに頼らない）。
- **再計算トリガ**: タワー設置/撤去のたびに、(1) 経路を 1 回再計算、(2) **生存する全 creep が現在地から経路を引き直す**。各回 `Path recalcs` を +1。敵数上限を上げるほど 1 操作あたりの A\* 回数が増え、CPU 負荷になる。
- **設置拒否**: 仮設置 → A\* で疎通確認し、到達不能配置はロールバック。
- 2D 化の肝は **`OrthographicCamera(0, W, H, 0, -1000, 1000)`**（1ワールド単位=1px・原点左下・Y上向き）。ゲームロジックは画面座標（Y 下向き・他エンジンと同一定数）のまま保持し、描画同期時のみ `worldY = H - gameY` に変換。
- スプライトは **`THREE.Sprite`（中心アンカー）** + `renderOrder`（タイル<経路<base<タワー<creep<弾<fx）、`depthTest:false`。
- **グリッドタイル（最大510枚）は基準実装と同様にプールした Sprite で描く**（数千 entity を作らない）。creep・弾・spark も同様にプール再利用。
- 経路ハイライトは薄い `PlaneGeometry`（セル塗り）+ `THREE.Line`（中心線）。経路更新時のみ再構築し、`geometry.dispose()` でリーク回避。
- 決定的擬似乱数 `mulberry32`（`Math.random` 不使用）。HUD は HTML オーバーレイ。比較指標は **Path recalcs**。

## Codex / AI コーディング所感
- 訓練データは最大級で API 自体は安定して書ける。3D サンプルは特に強い。
- 罠: ① ortho カメラの引数順と Y 向き（素直に書くと上下反転・カリング欠落）。② r150 以降の **ESM/importmap 必須**。③ 大量スプライトの `material/texture.dispose()` 漏れ。④ 「タイルを一枚ずつ Mesh で大量生成」しがちなので**プール方針を明示**。⑤ 経路探索を自前で持つ意図は指示が要る。
