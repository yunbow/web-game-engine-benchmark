# タワーディフェンス（経路探索）— KAPLAY 版

テーマ6共通仕様（タワーディフェンス：A\* 経路探索 × 多数ユニット追従・性能比較用）の **KAPLAY** 実装。
画像アセットが無くても単色図形フォールバックで必ず起動します。

使用バージョン: **KAPLAY 3001.0.19**（CDN: `unpkg.com/kaplay@3001.0.19/dist/kaplay.js`）。エンジン以外の追加 CDN は無し。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`6/`)をルートに HTTP サーバを立てて開く。

```bash
cd 6
python -m http.server 8000
# → http://localhost:8000/KAPLAY/
```
`file://` 直開きは画像が CORS で読めない（図形フォールバックでは起動する）。

## 操作
- **左クリック**: 通路タイルにタワー設置（25 Gold）。経路を塞ぐ配置は不可。
- **右クリック**: タワー撤去（返金なし・経路再計算）。
- `+` / `-`（テンキー +/- も可）: 同時出現する敵数の上限を ±10（下限10・上限500）。**性能比較の主軸**。
- `R`: リスタート（マップ・ゴールド・ライフ初期化）。

## 共通仕様の数値（全エンジン同一）
- キャンバス **960 x 540**、タイル **32x32**、グリッド **30 x 17**（決定的生成・固定シード `20250615`）。
- 敵: 半径10 / 70 px/s / HP30、スポーン間隔 0.5s。タワー: 射程96 / 連射0.6s / 弾10dmg / 弾速320。
- 初期 Gold 120 / Lives 20 / Score 0。撃破で Gold +5・Score +10。
- 敵数上限 初期30、±10、下限10・上限500。

## 実装メモ（テーマ6の要点）
- **A\* は自前実装**（4方向・コスト1・マンハッタン距離ヒューリスティック）。KAPLAY に経路探索の組み込みは無いが、仮にあっても**使わない**——本テーマの観測対象は「経路再計算回数」なので、全エンジンで同一の自前 A\* を共有する。
- **再計算トリガ**: タワー設置/撤去のたびに、(1) スタート→ゴールの経路を 1 回再計算、(2) **生存する全 creep が現在地から経路を引き直す**。各回 `Path recalcs` を +1 する。敵数上限を上げるほど、1 回のタワー操作で走る A\* 回数が増え、CPU 負荷として効く（= 観測の核）。
- **設置拒否**: 仮設置 → A\* で疎通確認し、ゴール到達不能になる配置はロールバックして拒否。
- 座標系は KAPLAY native（**Y 下向き・原点左上 = グリッド画面座標と一致**）で変換不要。
- 描画・ループ・入力は KAPLAY 機構（`add` / `onUpdate` / `dt()` / `onMousePress` / `onKeyPress` / `mousePos`）。グリッドタイル（最大510枚）・creep・弾・spark は**プール再利用**（毎フレームの生成破棄を避ける）。
- 決定的擬似乱数 `mulberry32`（`Math.random` 不使用）でマップを全エンジン共通化。
- HUD は HTML オーバーレイ（`#hud`）に **FPS / Enemies(生存/上限) / Towers / Projectiles / Path recalcs / Path len / Gold / Lives / Score** を表示。比較指標は **Path recalcs**。

## Codex / AI コーディング所感
- ミニマル API で「シーン構築〜ループ〜入力」を一発生成しやすい。マウスの左右判定は `onMousePress('left'/'right')` で素直。
- 罠: ① キー名が独自（`'='`/`'kpadd'`/`'minus'`/`'r'`）。② `mousePos()` は KAPLAY 内座標を返すため CSS スケール補正は自前不要だが、`getBoundingClientRect` 方式と混同しがち。③ 「経路探索を自前で持ち、再計算回数を数える」意図は明示指示が要る（エンジンに頼らせない）。
