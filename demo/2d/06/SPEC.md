# テーマ6 共通仕様 ― タワーディフェンス（経路探索 × 多数ユニット追従）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- グリッド上のタワーディフェンス。敵がスタート地点から経路を辿って自陣（ゴール）へ向かい、タワーが射程内の敵を撃つ。
- **目的**: 性能比較。**グリッド経路探索（A\*/BFS）＋ 多数ユニットの経路追従更新 ＋ タワー射撃**を同時に回した性能を測る
  （テーマ1〜5の「描画/自前衝突」中心に対し、本テーマは **CPU側のAIロジック（経路探索）と多数ユニットの追従** を主軸にする）。
- 性能比較の核は「**同時に経路追従している敵ユニット数**」と「**経路の再計算回数**」。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。タイルサイズ **32x32**。
- グリッド: **30 x 17 タイル**（= 960 x 544 px）。配列で定義。決定的に生成（固定シード）。
  - `0=通路(path/buildable)`, `1=壁(wall, 配置・通行不可)`。
  - **スタート**: 左端中央のタイル。**ゴール**: 右端中央のタイル。
  - 初期マップは外周を壁で囲み（スタート/ゴールの開口部のみ空ける）、内部に決定的な障害物（壁ブロック）を散らす。
- 経路は**グリッドのA\*（4方向, コスト1, マンハッタン距離ヒューリスティック）**でスタート→ゴールを毎回計算する。
  - 敵は**タワーの置かれたタイルを通行不可**として扱う。タワー設置/撤去で経路が変わるため**再計算**する。
  - ゴールへ到達不能になる配置は禁止（設置を拒否し、必ず経路が残るようにする）。

## 操作
- **左クリック**: 通路タイルにタワーを設置（コスト 25 Gold）。経路を塞ぐ配置は不可（設置失敗）。
- **右クリック**: タワーを撤去（返金なし。経路を再計算）。
- `+` / `-`: **同時出現する敵数の上限（ウェーブ規模＝負荷）を増減**。性能比較の主軸。
- `R`: リスタート（マップ・ゴールド・ライフ初期化）。

## ルール / 数値（全エンジン共通）
- 時間はデルタタイム基準（px/s, 秒）。
- **敵 (creep)**: 半径 **10**（描画直径 ~24）、移動 **70 px/s**、HP **30**。
  - 経路（タイル中心の列）に沿って線形に進む。次のウェイポイントへ向かい、到達したら次へ。
  - **タワー設置/撤去で経路が変わったら、各敵は現在地から再計算した経路に追従し直す**（再計算カウンタ +1）。
  - ゴール到達で消滅し **Lives -1**。HP0 で撃破 → **Gold +5, Score +10**、`hit_spark` 表示。
- **タワー**: 射程 **96 px**、連射間隔 **0.6 s**、弾ダメージ **10**。
  - 毎フレーム射程内の**最も進行度が高い（ゴールに近い）敵**を狙い、`projectile` を発射。
  - 弾速 **320 px/s**、敵に当たると消滅しダメージを与える（簡易: 弾と敵中心の距離 < 敵半径+弾半径6）。
- **ウェーブ**: 敵は左スタートから**一定間隔 0.5s**でスポーンし、画面内の生存数が上限（負荷値）未満のときのみ供給される
  （= 常に上限付近の敵が経路追従している状態を維持。ベンチを安定させる）。スポーン位置・初期ステータスは決定的。
- **資源**: 初期 **Gold 120 / Lives 20 / Score 0**。Lives 0 で **GAME OVER**（`R` で再開）。
  - ベンチ継続性のため、GAME OVER 表示中も `+`/`-` での負荷比較は維持してよいが、基本は `R` で再開。
- 物理エンジンは使わない（経路追従・弾の直進・距離判定はすべて自前）。

## 敵数（負荷）
- 同時出現上限 初期 **30** 体、`+`/`-` で **±10**（下限 **10**・上限 **500**）。
  - 上限を増やすほど「同時に経路追従＆射撃対象になる敵」が増える。タワー設置のたびに**全敵が一斉に再計算**するため、
    再計算コスト（A\*）と追従更新コストの両方が負荷として効く。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Enemies`（生存数 / 上限）/ `Towers`（設置数）/ `Projectiles`（生存弾数）
- `Path recalcs`（経路再計算の累計回数）/ `Path len`（現在のスタート→ゴール経路のタイル長）
- `Gold` / `Lives` / `Score`
- 画面下に操作ヒント行（左クリック=設置 / 右クリック=撤去 / +/-=敵数 / R=再開）。
