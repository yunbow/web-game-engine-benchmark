# タワーディフェンス（経路探索）— A-Frame 版

テーマ6共通仕様（タワーディフェンス：A\* 経路探索 × 多数ユニット追従・性能比較用）の **A-Frame** 実装。
画像アセットが無くても単色図形（canvas）フォールバックで必ず起動します。

使用バージョン: **A-Frame 1.7.0**（CDN: `aframe.io/releases/1.7.0/aframe.min.js`）。内部 three.js は `AFRAME.THREE`。エンジン以外の追加 CDN は無し。

## 起動方法
画像を `../assets/` から読むため、テーマフォルダ(`6/`)をルートに HTTP サーバを立てて開く。

```bash
cd 6
python -m http.server 8000
# → http://localhost:8000/A-Frame/
```
`file://` 直開きは画像が CORS で読めない（図形フォールバックでは起動する）。

## 操作
- **左クリック**: 通路タイルにタワー設置（25 Gold）。経路を塞ぐ配置は不可。
- **右クリック**: タワー撤去（返金なし・経路再計算）。
- `+` / `-`（テンキー +/- も可）: 同時出現する敵数の上限を ±10（下限10・上限500）。**性能比較の主軸**。
- `R`: リスタート。

## 共通仕様の数値（全エンジン同一）
- キャンバス **960 x 540**、タイル **32x32**、グリッド **30 x 17**（決定的生成・固定シード `20250615`）。
- 敵: 半径10 / 70 px/s / HP30、スポーン間隔 0.5s。タワー: 射程96 / 連射0.6s / 弾10dmg / 弾速320。
- 初期 Gold 120 / Lives 20 / Score 0。撃破で Gold +5・Score +10。敵数上限 初期30、±10、下限10・上限500。

## 実装メモ（設計判断つき・テーマ6の要点）
- A-Frame は three.js 上の宣言的（entity-component）フレームワーク。**シーンは `index.html` に `<a-scene>` として宣言**し、ゲーム本体は登録した **`td-game` コンポーネント**が駆動（A-Frame の renderer / `tick` ループ / カメラ管理を利用）。
- **設計判断（重要）**: タイル(510)・creep(数百)・弾を「1 = 1 `<a-entity>`」にすると DOM/コンポーネント生成コストで破綻するため、**動的オブジェクトはコンポーネント内で `THREE.Sprite` を直接生成・管理**する（`this.el.object3D` に add）。グリッドタイルもプールした Sprite で描く。
- 2D 化のため `tick` で `sceneEl.camera` を **`OrthographicCamera(0,W,H,0,-1000,1000)`** に維持（A-Frame の既定 perspective カメラを上書き）。座標は `worldY = H - gameY` 変換。`renderer.setPixelRatio(1)` で DPR を 1 固定。
- **A\* は自前実装**（4方向・コスト1・マンハッタン距離）。本テーマの観測対象は「経路再計算回数」なので、全エンジンで同一の自前 A\* を共有する（A-Frame/three の組み込みに頼らない）。
- **再計算トリガ**: タワー設置/撤去のたびに、(1) 経路を 1 回再計算、(2) **生存する全 creep が現在地から経路を引き直す**。各回 `Path recalcs` を +1。敵数上限を上げるほど 1 操作あたりの A\* 回数が増える。
- **設置拒否**: 仮設置 → A\* で疎通確認し、到達不能配置はロールバック。
- マウス入力は `sceneEl.canvas` に `pointerdown`（`event.button` で左右判別）と `contextmenu` 抑止を登録。キーは `keydown`。決定的擬似乱数 `mulberry32`。HUD は HTML オーバーレイ。比較指標は **Path recalcs**。

## Codex / AI コーディング所感
- **HTML 宣言的**なのでシーンの骨格は AI が書きやすい。一方、2D アーケード/TD はミスマッチが大きい。
- 罠: ① 大量オブジェクトを素直に `<a-entity>` 化すると破綻（THREE 直管理が必須）。② 2D 用 ortho カメラは既定が無く `sceneEl.camera` 差し替えという定石を知らないと perspective のまま実装しがち。③ `tick(time, dtMs)` の dt が **ms 単位**。④ マウス座標はキャンバスの `getBoundingClientRect` で CSS スケール補正が要る。⑤ 経路探索は自前で持ち再計算回数を数える方針を明示指示する必要がある。
