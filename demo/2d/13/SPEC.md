# テーマ13 共通仕様 ― 大量テキスト / UI 描画（動的テキスト・グリフ描画）

全エンジン（Phaser4 / Babylon.js / PixiJS / LittleJS / Godot）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。

## ゲーム内容
- 画面いっぱいに多数のテキストラベルが流れ、各ラベルの文字列が刻々と更新される「システムログ / データダッシュボード」風のデモ。
- **目的**: 性能比較。**多数のテキストオブジェクト ＋ 動的な文字列更新（再レイアウト/グリフのテクスチャ再生成）**のスループットを測る。
  - テーマ1〜12は HUD が数行のテキストのみで、**テキスト描画自体は誰も負荷にしていない**。テキスト/グリフ描画は各エンジンで実装が大きく異なり、**既知のパフォーマンスの崖**でもある。本テーマはこの未計測軸を埋める。
- 性能比較の核は「**画面上のテキストオブジェクト数**」と「**毎フレーム文字列を更新する（動的）か否か**」のコスト差。

## テキスト機構（エンジンごとに“素直な選択”を使う ＝ これ自体が比較対象）
- 各エンジンの**標準テキスト機構**を使う。何を使ったか・崩れ始める点を README に記載する。
  - **Phaser4**: `this.add.text`（Canvas ベースの Text）。任意で `bitmapText` 比較に言及。
  - **PixiJS**: `PIXI.Text`（Canvas/HTML テキスト）。任意で `BitmapText` 比較に言及。
  - **Babylon.js**: GUI `AdvancedDynamicTexture` の `TextBlock`、または DynamicTexture へのテキスト描画。採用方式を明記。
  - **LittleJS**: `drawTextScreen`/`drawText`（Canvas2D テキスト）。
  - **Godot**: `Label` ノード、または `_draw` の `draw_string`（既定フォント）。
- 外部フォントアセットは**不要**（各エンジンの既定/システムフォントで可）。

## 画面・基本設定
- キャンバス: **960 x 540**（固定）。背景は暗色（`#0b0d16` 相当）。
- 時間はデルタタイム基準。

## テキストアイテム（負荷の主役）
- `N` 個のテキストアイテムが画面内を**決定的に流れる**（下方向スクロール or 左右バウンド。固定シードで初期位置・速度・色・基準文字列を決定）。
- 各アイテムの文字列は短め（**8〜20文字程度**）。例: `"OBJ#0042 v=137"` のような「ラベル＋ライブ更新される数値」。色・サイズはアイテムごとに決定的に割当。
- 画面外へ出たアイテムは反対側から回り込む（ラップ）。配置・動きは決定的で、無入力でも回り続ける。
- 別途、画面隅に**常時更新される複数行の統計パネル**（フレーム番号・各種カウンタ等）を1つ置き、多行の動的テキスト再レイアウトも踏ませる。

## 動的更新（コストの主軸）
- **Update = ON（既定）**: 毎フレーム、各アイテムの文字列を更新する（数値部分を再計算して文字列を作り直す）。
  → テキストの**再レイアウト/グリフのテクスチャ再生成**が毎フレーム走る（重い経路）。
- **Update = OFF**: 文字列は固定のまま位置だけ動かす（多くのエンジンは内部キャッシュが効き軽くなる）。
- `U` キーで Update ON/OFF を切替（この差が比較の核の一つ）。

## 操作
- `+` / `-`: **テキストアイテム数（負荷）を増減**。性能比較の主軸。
- `U`: 文字列の毎フレーム更新 ON/OFF 切替。
- `B`（任意・対応エンジンのみ）: 標準テキスト ⇄ ビットマップ/キャッシュ系テキストの切替（実装した場合のみ。未対応なら無効でよく、README に明記）。
- `R`: リセット。

## テキスト数（負荷）
- `N` 初期 **200**、`+`/`-` で **±100**（下限 **0**・上限 **5000**）。
  - 増やすほど「描画するテキストオブジェクト数」と（Update ON 時の）「文字列再生成・グリフ処理」が増える。
    Canvas テキスト（毎回ラスタライズ）と、ビットマップ/キャッシュ系で `FPS` の落ち方に差が出る。

## アセット
- **本テーマはアセット不要**（各エンジンの既定/システムフォントを使用）。`assets/.gitkeep` のみ。
- 外部ビットマップフォントは使わない（`B` のビットマップ比較を実装する場合も、エンジン内蔵/動的生成の範囲で）。

## 共通HUD（必須）
- `FPS`（実測, 移動平均）
- `Texts`（テキストアイテム数 現在 / 上限）
- `Chars`（概算の総グリフ数 = アイテム数 × 平均文字数。統計パネル分を含めてよい）
- `Render`（使ったテキスト機構: 例 "Canvas Text" / "BitmapText" / "TextBlock" / "draw_string"）
- `Update`（dynamic = 毎フレーム文字列更新 / static）
- 画面下に操作ヒント行（+/-=テキスト数 / U=動的更新 / B=機構切替 / R=リセット）。
