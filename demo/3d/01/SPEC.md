# 3D テーマ1 共通仕様 ― インスタンス小惑星フィールド（3Dシューティング）

全ライブラリ（three.js / Babylon.js / PlayCanvas / A-Frame）で**同一ゲーム**を実装し、性能を横並び比較するための共通仕様。
**比較の主軸 = 同一メッシュの大量インスタンス描画スループット**（GPUインスタンシング）。

> この SPEC.md が唯一の正。速度・数量・当たり判定方式・カメラ設定・座標系は4ライブラリで**完全一致**させる。
> 1ライブラリだけ値を変えると比較が無効になる。

## ゲーム内容
- 3D空間を奥（-Z）から手前（+Z）へ流れてくる大量の小惑星を、自機が避けつつ撃つ3Dシューティング。
- **目的は性能比較**: 同一の低ポリメッシュを数千〜数万体インスタンス描画し、毎フレーム位置更新する負荷で FPS を測る。

## 描画バックエンド（公平比較の前提）
- **全ライブラリ WebGL2 で統一**。WebGPU は使わない（three.js も `WebGLRenderer`、Babylon は既定の WebGL2、PlayCanvas は `DEVICETYPE_WEBGL2`）。
- これによりバックエンド差を排除し、**各ライブラリのインスタンシング機構そのもの**を比較する。

## 座標系・カメラ（4ライブラリで一致させる）
- **右手系・Y軸上向き**（three.js / Babylon(右手系に設定) / PlayCanvas / A-Frame すべてこの向きに統一）。
  - Babylon.js は既定が左手系のため **`scene.useRightHandedSystem = true`** を設定すること。
- 透視投影カメラ: **fov 60°（垂直）/ near 0.1 / far 2000**。
- カメラは自機の後方やや上に固定追従: **自機位置 + (0, 6, 22)** に置き、**自機位置 + (0, 2, 0)** を注視。
- 描画サイズ: **960 x 540（16:9 固定）**。背景は暗い宇宙色 `#05060a`。
- 環境光 + 平行光1灯（方向 `(-0.5, -1, -0.3)` 正規化, 白, 強度1.0）。影は無し（T5で扱う）。

## プレイ空間（ワールド単位）
- 自機は **z = 0 の平面**上を移動。可動範囲 **x ∈ [-60, 60] / y ∈ [-34, 34]**（範囲外はクランプ）。
- 小惑星の出現〜消滅: **z = -1200（奥）で出現 → +Z 方向へ前進 → z > 30（カメラ後方）で消滅し、奥へ再出現（リサイクル）**。
- 弾は自機から **-Z 方向**へ発射。**z < -1200** で消滅。

## 操作
- 移動: **矢印キー / WASD**（自機は x/y 平面を8方向移動、範囲クランプ）。移動速度 **60 u/s**。
- 発射: **常時オート連射（150msごと）**、弾速 **400 u/s（-Z方向）**。
- **`+` / `-`（および `]` / `[`）: 小惑星の同時数を増減**（＝比較の主軸負荷）。
- `P`: オートプレイのトグル（任意・既定OFF。無人ベンチ用に自機が決定的に左右往復）。

## ルール / 数値（全ライブラリ共通）
- **小惑星数**: 初期 **2000**。`+`/`-` で **±1000**（**最小 1000 / 最大 50000**）。
  - 数を増やしたとき新規ぶんは奥（z=-1200付近）に追加、減らすぶんは末尾から除去。
- 小惑星の前進速度: 個体ごと **80〜160 u/s**（決定的に分配）。半径 **2.0〜5.0**（決定的）。見た目用に毎フレーム自転。
- 弾: 半径 **0.5**、連射間隔 **150ms**、同時上限 **64発**（プール）。
- 自機: 半径 **2.0**（当たり判定）。被弾後 **1.0s 無敵**（点滅は任意）。
- **当たり判定はすべて自前の球判定（中心間の平方距離 ≤ 半径和の2乗）**。物理エンジン不使用。
  - 弾 × 小惑星: ヒットで小惑星を奥へリサイクル、弾を回収、**Score +10**。
  - 小惑星 × 自機: ヒットで **HP -1**（無敵中は無効）、その小惑星を奥へリサイクル。
- HP 初期 **3**。HP 0 でゲームオーバー表示（自機停止・小惑星は流れ続ける）。`R`で再開（任意）。

## 決定的生成（必須・`Math.random` 不使用）
- 小惑星の初期位置・速度・半径・自転は**シード付き疑似乱数**で生成（例: mulberry32, seed=`0x9e3779b9`）。
- これにより4ライブラリで同一の小惑星配置・無人オートプレイの再現ベンチが可能。

## インスタンシング機構（各ライブラリの“素直な選択”）
- 小惑星も弾も**1メッシュをインスタンス描画**し、毎フレーム per-instance 変換行列（位置/回転/スケール）を更新する。
- three.js: `THREE.InstancedMesh`（`setMatrixAt` + `instanceMatrix.needsUpdate`）。
- Babylon.js: **thin instances**（`thinInstanceSetBuffer("matrix", ...)` を毎フレーム更新）。
- PlayCanvas: **hardware instancing**（per-instance の行列 VertexBuffer + `meshInstance.setInstancing`）。
- A-Frame: three.js の `InstancedMesh` を**カスタムコンポーネント**で `el.object3D` に載せる（A-Frameの宣言性は維持しつつ大量描画はインスタンスで）。
- メッシュ形状: **低ポリ正二十面体（icosahedron, detail=0 ＝20面）** を小惑星に、**小球（低分割 sphere）** を弾に使用。

## アセット
- **T1 は画像/GLB を使わずプリミティブのみで起動**（図形フォールバックがそのまま本実装）。後続テーマでGLB/テクスチャを追加する。
- 個体差はインスタンスごとの**色ティント**（決定的）で表現してよい。

## 共通HUD（必須・性能比較の要）
画面左上にHTMLオーバーレイ（A-Frame含むJS系すべてHTMLオーバーレイ）で表示:
- `FPS`（実測・移動平均）
- `Objects`（画面上の小惑星 + 弾の合計数）
- `Score` / `HP`
- `Asteroids`（現在の小惑星上限設定値）
- 可能なら `Draws`（draw call 数）/ `Tris`（三角形数）も表示（各レンダラの統計APIから取得・ベストエフォート）。

## 成果物
- 各ライブラリフォルダ: `index.html`（＋必要なら `game.js`）。**CDNでライブラリ読込**。ローカルHTTPサーバで起動。
- 各フォルダに `README.md`（起動方法・使用バージョン・実装メモ・AIコーディング生成のしやすさ所感）。
