# 3D テーマ10（T10）共通仕様 ― 大量レイキャスト（LIDAR スキャナ）

全ライブラリ（three.js / Babylon.js / PlayCanvas / A-Frame）で**同一仕様**を実装し、性能を横並び比較するための共通仕様。
**比較の主軸 = 毎フレームの大量レイキャスト（レイ×ターゲットメッシュの交差判定）のスループット**。

> この SPEC.md が唯一の正。レイ数・ターゲット配置・スキャナ・カメラ・HUD を4ライブラリで揃える。
> **レイ-メッシュ交差は各エンジンの機構を使う**（three=Raycaster / Babylon=Ray.intersectsMesh / A-Frame=three Raycaster /
> PlayCanvas=組み込みのレイ-メッシュが無いため自前のレイ-AABB）。その「統合と同一負荷での FPS」が比較対象。

## ゲーム内容
- 中心のスキャナから**毎フレーム N 本のレイ**を全方位に放ち、各レイが**最近のターゲットへの当たり**を求め、当たり点に印を表示する（LIDAR/スキャン可視化）。
- **目的は性能比較**: `+`/`-` でレイ数を増減し、N×M のレイキャスト＋当たり点描画のコストで FPS を測る。

## 描画バックエンド
- **全ライブラリ WebGL2 で統一**。当たり点マーカーは**インスタンス描画**。

## 画面・カメラ
- 描画サイズ **960 x 540（16:9 固定）**。背景 `#05080d`。**Y軸上向き**。
- カメラは**緩やかに自動周回**（決定的）: 半径 **55**、高さ **20**、角速度 **0.15 rad/s**、注視 **(0,0,0)**。fov 55° / near 0.5 / far 1000。
- 環境光 + 平行光1灯。

## ターゲット（決定的・`Math.random` 不使用）
- ターゲット数 **M = 120**（固定）。原点を囲む**半径 28 の球殻上**にフィボナッチ球分布で配置（決定的）:
  - i=0..M-1: `y = 1 - 2*(i+0.5)/M`, `r = sqrt(1-y*y)`, `θ = i * 2.399963`（黄金角）, 方向 `(r cosθ, y, r sinθ)`、位置 = 方向 × 28。
- 形状: 一辺 **4** の**軸整列（無回転）ボックス**（インスタンス描画）。色は中明度。
  - ※軸整列必須: PlayCanvas は自前のレイ-AABB を使うため、回転ボックスだと three/Babylon のレイ-メッシュ結果とズレる。全エンジンで無回転に揃える。

## スキャナ / レイ（決定的）
- スキャナ原点: 中心付近で微小に上下（決定的）: `origin = (0, sin(t*0.7)*2, 0)`。小さな自発光マーカーを表示。
- **レイ数 N**: 初期 **1500**。`+`/`-`（`]`/`[`）で **±1500**（**最小 500 / 最大 15000**）。**比較の主軸**。
- レイ方向: **フィボナッチ球**で N 本を全方位に均等分布（決定的・毎フレーム同分布、`t` でゆっくり回転させてもよい）。
- 各レイは origin から方向へ、**M 個のターゲットとの交差**を判定し、**最も近い当たり点**を採用。レイ長上限 **far = 200**。

## 当たり点の可視化
- 当たったレイの**最近交差点**に小さな印（半径 ~0.4 の球）をインスタンス描画。当たらなかったレイは印なし。
- 当たり点数（Hits）を HUD に出す。

## 操作
- `+` / `-`（`]` / `[`）: レイ数の増減。**比較の主軸**。
- `R`: レイ数を初期値に戻す。
- カメラ自動周回・スキャナ自動（無人ベンチ可）。

## 共通HUD（必須）
画面左上にHTMLオーバーレイ:
- `FPS`（実測・移動平均）
- `Objects`（ターゲット数 = M）
- `Rays`（現在のレイ数設定 N）
- `Hits`（当たったレイ数）
- `Draws` / `Tris`（描画統計・ベストエフォート）。

## アセット
- **画像/GLB不使用・プリミティブ（box/sphere）のみ**。色は固定。
