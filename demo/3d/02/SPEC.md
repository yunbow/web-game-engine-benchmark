# 3D テーマ2（T3）共通仕様 ― 3D剛体物理（箱タワー崩し）

全ライブラリ（three.js / Babylon.js / PlayCanvas / A-Frame）で**同一仕様**のゲームを実装し、性能と統合相性を横並び比較するための共通仕様。
**比較の主軸 = 本物の3D剛体物理エンジン統合と剛体数スケール**。2Dテーマ7（matter-js横並び）の3D版。

> この SPEC.md が唯一の正。数値（重力・寸法・数量・反発/摩擦・発射・カメラ）は4ライブラリで一致させる。

## 重要な前提（物理は「自前実装しない」唯一の比較対象）
- T1や2Dの多くと違い、**本テーマは各ライブラリの“素直な物理エンジン”を必ず使う**（自前物理は不可）。
  - **three.js → Rapier3d**（`@dimforge/rapier3d-compat`, CDN/WASM）
  - **Babylon.js → Havok**（`@babylonjs/havok`, 公式プラグイン, CDN/WASM）
  - **PlayCanvas → ammo.js(Bullet)**（`rigidbody`/`collision` コンポーネント, CDN/WASM）
  - **A-Frame → physics system**（`aframe-physics-system` の ammo ドライバ等, CDN）
- **ソルバが各エンジンで異なるため、箱の崩れ方（軌道）は完全一致しない**。これは想定どおりで、比較対象は「統合の手間・APIの素直さ・同一剛体数でのFPS」。**初期レイアウト・数値は一致させる**。

## ゲーム内容
- 地面の上に箱を格子状に積んだ「壁／タワー」。一定間隔で**砲弾（球）が自動発射**されタワーに衝突、崩れていく。
- **目的は性能比較**: 剛体（箱）数を `+`/`-` で増やし、同一剛体数での FPS と挙動を観測する。

## 画面・座標・カメラ
- 描画サイズ **960 x 540（16:9 固定）**。背景は暗color `#0a0c10`。
- **Y軸上向き**。地面は y=0 の水平面。各エンジンの既定ハンドを使ってよい（左右手系の違いは本テーマの対称的レイアウトでは比較に影響しない）。
- カメラは固定: 位置 **(0, 14, 56)** から 注視点 **(0, 10, 0)**。fov 50° / near 0.1 / far 2000。
- 環境光 + 平行光1灯（上方やや手前から）。影は任意（重い場合OFF、4本で揃える）。

## 物理パラメータ（全ライブラリ共通）
- 重力: **(0, -20, 0)**。
- 箱: **2 x 2 x 2** の立方体。質量 **1**。反発(restitution) **0.1** / 摩擦(friction) **0.6**。
- 砲弾: 半径 **1.5** の球。質量 **8**。反発 **0.2** / 摩擦 **0.4**。
- 地面: 静的（無限/大判の床）。反発 **0.1** / 摩擦 **0.8**。
- 物理ステップは **固定 1/60s**（各エンジンの推奨に従いつつ、可能な限り 60Hz 固定）。

## タワー構築（決定的・`Math.random` 不使用）
- 剛体（箱）数 **N**: 初期 **200**。`+`/`-`（および `]`/`[`）で **±50**（**最小 20 / 最大 1500**）。
- レイアウト: 幅 **`cols = 20`** 列固定、行は `rows = ceil(N / cols)`。最終行は端数ぶんだけ置く。
  - 箱の中心 x = `(c - (cols-1)/2) * 2.05`（列 c=0..cols-1, 2.0幅 + 0.05隙間）。
  - 箱の中心 y = `1.0 + r * 2.02`（行 r=0..rows-1, 下段が地面接地）。
  - z = 0（1枚壁）。微小な初期傾き等は入れない（決定的）。
- `+`/`-` で N を変えたら**タワーを再構築**（全剛体を作り直し、砲弾もクリア）。

## 砲弾の発射（決定的）
- **2.0秒ごとに自動発射**。発射位置 **(0, 10, 40)**、初速 **(0, 2, -55)**（タワーへ向かって -Z、わずかに上向き）。
- 砲弾は最大 **8発**プール。`z < -60` または `y < -20` で消滅（リサイクル）。
- `Space`: 手動でも1発発射（任意）。

## 操作
- `+` / `-`（`]` / `[`）: 箱（剛体）数の増減 → タワー再構築。**比較の主軸**。
- `Space`: 砲弾を即発射（任意）。
- `R`: タワー再構築（リセット）。
- 自機操作は無し（無人で物理が進行＝オートベンチ可能）。

## スコア
- 箱の中心が **y < 0.5（地面近く＝崩れ落ちた）** に初めて達したらその箱を「撃破」とみなし **Score +10**（1箱1回）。

## 共通HUD（必須・性能比較の要）
画面左上にHTMLオーバーレイ:
- `FPS`（実測・移動平均）
- `Objects`（剛体合計 = 箱N + アクティブ砲弾数）
- `Score`
- `Bodies`（現在の箱数設定値 N）
- 可能なら `Draws` / `Tris`（各レンダラ統計・ベストエフォート）。

## アセット
- **画像/GLB 不使用・プリミティブ（box / sphere / plane）のみで起動**。色は固定（箱=石色、砲弾=赤、床=暗灰）。
