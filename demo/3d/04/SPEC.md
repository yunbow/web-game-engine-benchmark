# 3D テーマ4（T6）共通仕様 ― GPUパーティクル（魔法/噴水エフェクト）

全ライブラリ（three.js / Babylon.js / PlayCanvas / A-Frame）で**同一仕様**を実装し、性能を横並び比較するための共通仕様。
**比較の主軸 = パーティクル機構のスループット（粒子数スケール）と加算ブレンド描画**。2Dテーマ8（GPUパーティクルFX）の3D版。

> この SPEC.md が唯一の正。粒子数・寿命・速度域・重力・色・ブレンド・カメラ・HUD を4ライブラリで揃える。
> （各エンジンの粒子機構＝シェーダ駆動 / GPUParticleSystem / particlesystem は実装が異なるため、粒子の軌道は完全一致しない。
> 揃えるのは「粒子数・噴出の見た目の性質・加算発光・数値の意味」。）

## 描画バックエンド
- **全ライブラリ WebGL2 で統一**（WebGPU不使用）。**WebGL2 には compute シェーダが無い**ため、
  各エンジンの“素直な”粒子機構を使う:
  - **three.js**: `THREE.Points` + 自作 `ShaderMaterial`（**頂点シェーダで各粒子を時間から計算**＝GPU側アニメ・CPU毎フレーム更新なし）。
  - **Babylon.js**: `GPUParticleSystem`（WebGL2 の transform feedback による真のGPUパーティクル）。
  - **PlayCanvas**: `particlesystem` コンポーネント（GPU 加速パーティクル）。
  - **A-Frame**: three.js の `Points` + `ShaderMaterial` をカスタムコンポーネントで（three.js版と同方式）。

## ゲーム内容
- 原点付近から**上方へ噴き上がり重力で落ちる噴水状のパーティクル**を大量発生（魔法/炎風の加算発光）。
- **目的は性能比較**: 目標パーティクル数を `+`/`-` で増減し、機構のスループットで FPS を測る。

## 画面・カメラ
- 描画サイズ **960 x 540（16:9 固定）**。背景 `#05060a`（暗い宇宙）。**Y軸上向き**。
- カメラ固定: 位置 **(0, 8, 26)** / 注視 **(0, 5, 0)** / fov 55° / near 0.1 / far 2000。
- ライト不要（加算発光のため。粒子は自発光色）。

## パーティクル挙動（全ライブラリ共通の“見た目の性質”）
- **エミッタ＝原点 (0,0,0) 付近**から連続噴出（噴水）。
- 各粒子の初速: **上方向(+Y)を軸とする広めのコーン**内のランダム方向 × 速さ **4〜10 u/s**（決定的に分配）。
- 重力: **(0, -9, 0)**。粒子は放物線で上がって落ちる。
- **寿命 3.0s** でループ（寿命が尽きたらエミッタへ戻り再噴出＝定常噴水）。
- 色（加算ブレンド）: 誕生〜消滅で **明るい黄 `#fff1a8` → 橙 `#ff8a3d` → 赤紫へフェード**（寿命で輝度を落としアルファも 1→0）。
- 点サイズ: 誕生時最大→寿命で縮小。透視で遠いほど小さく。
- **ブレンド = 加算（additive）**、深度書き込みOFF（重なって発光が加算される）。

## 数値（共通）
- 目標粒子数 **N**: 初期 **20000**。`+`/`-`（`]`/`[`）で **±20000**（**最小 5000 / 最大 500000**）。
- three.js / A-Frame は最大数ぶんのバッファを確保し **描画範囲(drawRange)で N を制御**（再生成なし）。
  Babylon/PlayCanvas は各システムの容量(capacity/numParticles)を N に設定（必要なら再生成）。

## 決定的生成（`Math.random` 不使用）
- 各粒子の方向・速さ・寿命オフセット（位相）は**シード付き疑似乱数**（mulberry32, seed=0x9e3779b9）で決定的に与える。
- これにより無人で定常噴水が再現され、4ライブラリで粒子数あたりの負荷を比較できる。

## 操作
- `+` / `-`（`]` / `[`）: 目標粒子数の増減。**比較の主軸**。
- `R`: リセット。
- 入力なしで噴出は継続（無人ベンチ可）。

## 共通HUD（必須）
画面左上にHTMLオーバーレイ:
- `FPS`（実測・移動平均）
- `Objects`（描画中パーティクル数 = N）
- `Particles`（現在の目標粒子数設定 N）
- 可能なら `Draws`（draw call 数）/ `Points`（描画点数。`Tris` は点描画では無意味なので Points を出す）。

## アセット
- **画像不使用**。点スプライトはシェーダ内で円形・ソフトエッジを生成（フラグメントで距離に応じ discard / アルファ減衰）。
  Babylon/PlayCanvas が内蔵スプライトテクスチャを要する場合はコード生成（小さな放射状グラデを canvas で）でよい。
