# 3D テーマ3（T4）共通仕様 ― スキンドキャラ大群（glTF + スケルタルアニメ）

全ライブラリ（three.js / Babylon.js / PlayCanvas / A-Frame）で**同一仕様**を実装し、性能を横並び比較するための共通仕様。
**比較の主軸 = スキンドメッシュ（スケルタルアニメ）の大量再生スループット**。2Dにはスプライトのコマ送りしか無く、3D固有の比較軸。

> この SPEC.md が唯一の正。数値・配置・カメラ・HUD・アニメ設定は4ライブラリで一致させる。
> （スキニングの実装機構＝スケルトン更新・スキニング行列の計算はエンジンごとに異なるが、見える挙動・数値は揃える）

## ゲーム内容
- 地面の上に、**リギング済みglTFキャラ（歩行アニメ付き）を N 体**並べ、各個体が独立にアニメ再生する「群衆」デモ。
- **目的は性能比較**: 同時アニメキャラ数を `+`/`-` で増減し、AnimationMixer/スケルトン更新のコストで FPS を測る。

## 描画バックエンド
- **全ライブラリ WebGL2 で統一**（WebGPU不使用）。スキニングは頂点シェーダ（GPUスキニング）。

## アセット（共有 glTF）
- `../assets/CesiumMan.glb`（Khronos サンプル、リギング済み・歩行アニメ1本同梱、約 438KB）を**全ライブラリ共通**で読み込む。
  - 4ライブラリとも glTF/GLB をネイティブ読込可（three=GLTFLoader / Babylon=SceneLoader / PlayCanvas=glb container / A-Frame=gltf-model）。
- **GLB 読込失敗時は図形フォールバックで必ず起動**: 各キャラを「上下に弾むカプセル/箱」に置換し、同数を同配置でアニメ（※スキニングは再現されない旨をHUDかREADMEに明記）。

## 座標系・カメラ
- 描画サイズ **960 x 540（16:9 固定）**。背景 `#10131a`。**Y軸上向き**。地面 y=0。
- カメラ固定: 位置 **(0, 12, 26)** / 注視 **(0, 1.5, 0)** / fov 50° / near 0.1 / far 2000。
- 環境光 + 平行光1灯（上方やや手前）。影は無し（4本で揃える）。

## キャラ配置（決定的・`Math.random` 不使用）
- キャラ数 **N**: 初期 **50**。`+`/`-`（`]`/`[`）で **±25**（**最小 10 / 最大 1000**）。
- グリッド配置: `cols = ceil(sqrt(N))`、行 `rows = ceil(N / cols)`。中心揃え、**間隔 2.2**。
  - 個体 i: `c = i % cols`, `r = floor(i / cols)`、`x = (c - (cols-1)/2) * 2.2`、`z = (r - (rows-1)/2) * 2.2`、`y = 0`。
- モデルが大きすぎ/小さすぎる場合、**全体スケールを統一値で調整**してよいが、4ライブラリで同じスケールにする（基準: CesiumMan を約 1.6〜1.8 ワールド単位の身長に収める。three.js 参照実装の scale 値に合わせる）。
- キャラはその場で歩行アニメを再生（**前進はしない＝歩行サイクルをその場で回す**）。向きは全個体同一（+Z 向き）。

## アニメーション（決定的）
- 同梱の歩行クリップを **ループ再生**。
- 個体ごとに **再生速度 timeScale ∈ [0.8, 1.2]** と **開始位相オフセット ∈ [0, クリップ長)** を**シード付き疑似乱数**（mulberry32, seed=0x9e3779b9）で決定的に与え、群衆が同期しないようにする。
- 各個体が独立した mixer / animation group / anim を持つ（＝N体ぶんのスケルトン更新が毎フレーム走るのが負荷の主役）。

## 操作
- `+` / `-`（`]` / `[`）: キャラ数の増減 → 群衆を再構築。**比較の主軸**。
- `R`: 再構築（リセット）。
- 入力なしでアニメは進行（無人ベンチ可）。

## 共通HUD（必須）
画面左上にHTMLオーバーレイ:
- `FPS`（実測・移動平均）
- `Objects`（描画中キャラ数 = N）
- `Chars`（現在のキャラ数設定 N）
- 可能なら `Draws`（draw call 数）/ `Tris`（三角形数）。
- GLB 読込に失敗してフォールバック中はその旨（例 `(fallback: no skin)`）を表示。
