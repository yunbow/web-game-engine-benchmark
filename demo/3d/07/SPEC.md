# 3D テーマ7（T7）共通仕様 ― ボクセルチャンク再生成（動的メッシュ再構築/再アップロード）

全ライブラリ（three.js / Babylon.js / PlayCanvas / A-Frame）で**同一仕様**を実装し、性能を横並び比較するための共通仕様。
**比較の主軸 = 毎フレームの頂点バッファ再構築と GPU 再アップロードのスループット**。2Dテーマ12（フォーリングサンド＝毎フレーム全面テクスチャ転送）の3D版で、転送対象が「テクスチャ」ではなく「ジオメトリ（頂点バッファ）」。

> この SPEC.md が唯一の正。チャンク構成・セル数・波・カメラ・HUD を4ライブラリで揃える。

## ゲーム内容
- ブロック状の地形（ボクセル風）を**チャンク単位のメッシュ**で表示。時間変化する波で各セルの高さが**毎フレーム変わり**、
  各チャンクのメッシュを**毎フレーム作り直して GPU に再アップロード**する。
- **目的は性能比較**: `+`/`-` で**チャンク数（描画距離）**を増減し、毎フレームのメッシュ再構築＋転送コストで FPS を測る。

## 描画バックエンド
- **全ライブラリ WebGL2 で統一**。各エンジンの動的メッシュ更新機構を使う:
  - three.js: `BufferGeometry`（事前確保した `position/normal/color` 属性を毎フレーム書き換え `needsUpdate=true`）。
  - Babylon.js: `VertexData` を `mesh.updateVerticesData(...)`（updatable mesh）で更新。
  - PlayCanvas: `pc.Mesh` の `setPositions/setNormals/setColors/setIndices` → `mesh.update()`。
  - A-Frame: three.js の `BufferGeometry`（three.js版と同方式）をカスタムコンポーネントで。

## 画面・カメラ
- 描画サイズ **960 x 540（16:9 固定）**。背景 `#0b1016`。**Y軸上向き**。
- カメラ固定: 位置 **(0, 60, 95)** / 注視 **(0, 4, 0)** / fov 55° / near 0.5 / far 2000。
- 環境光 + 平行光1灯（上方やや手前、フラットシェーディングのブロック感を出す）。

## チャンク / セル構成（決定的・`Math.random` 不使用）
- **セルサイズ CS_SIZE = 2**（ワールド単位）。1チャンク = **CS × CS = 12 × 12 = 144 セル**。1チャンクの一辺 = 24。
- **チャンク数 NC × NC**: 初期 **NC = 4（= 16 チャンク）**。`+`/`-`（`]`/`[`）で **NC ±1**（**最小 2（4チャンク）/ 最大 8（64チャンク）**）。**比較の主軸**。
  - チャンク群は中心揃え。チャンク (Ci, Cj) の原点 ox/oz は全体が原点中心になるよう配置。
- 各セルの**グローバル列座標** `gx = Ci*CS + cx`、`gz = Cj*CS + cz`（チャンクをまたいで連続）。

## 高さの波（毎フレーム・決定的）
- 各セルの高さ（整数ステップでブロック感）:
  `h = 1 + floor( (sin(gx*0.25 + t) + cos(gz*0.25 + t*0.8) + 2) * 2 )`（範囲 概ね 1〜9, `t` は経過秒）。
- これにより全チャンクが毎フレーム形状変化 → 全チャンクのメッシュを毎フレーム再構築・再アップロードする（負荷の主役）。

## メッシュ生成（各セル）
- 各セルは **上面1枚 + 側面4枚（底 y=0 まで）の計5クアッド**（＝ブロック柱）。隣接カリングはせず一律生成（実装を4ライブラリで揃えるため）。
  - 非インデックスでよい（1セル = 5クアッド = 30頂点）。法線は面ごと（フラットシェーディング）。
- **色は高さで決定**（低=緑 → 中=茶 → 高=白 のグラデ）。セル単位の単色でよい。
- 1チャンクの頂点数は固定（144セル × 30 = 4320頂点）。**バッファは事前確保し、毎フレーム中身だけ書き換える**（再アロケートしない）。

## 操作
- `+` / `-`（`]` / `[`）: チャンク数 NC の増減。**比較の主軸**。NC 変更時はチャンク群を作り直す。
- `R`: NC を初期値に戻す。
- カメラ固定・波は自動進行（無人ベンチ可）。

## 共通HUD（必須）
画面左上にHTMLオーバーレイ:
- `FPS`（実測・移動平均）
- `Objects`（チャンク数 = NC×NC）
- `Chunks`（現在の NC×NC、または `NC²`）
- `Draws`（draw call 数 ≒ チャンク数）
- `Tris`（三角形数 = チャンク数 × 144セル × 10三角）

## アセット
- **画像/GLB不使用・コード生成ジオメトリのみ**。色はコード生成。
