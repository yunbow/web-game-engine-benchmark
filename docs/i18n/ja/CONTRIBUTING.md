# Web Game Engine Benchmark への貢献

ご興味をお持ちいただきありがとうございます。

## 貢献の方法

### Issue の報告

バグや提案は [GitHub Issues](https://github.com/yunbow/web-game-engine-benchmark/issues) へ報告してください。
デモの不具合を報告する際は、テーマ番号・エンジン名・使用したブラウザを明記してください。

### プルリクエスト

1. リポジトリをフォークする
2. フィーチャーブランチを作成する
3. 変更を行う
4. わかりやすいメッセージでコミットする
5. プルリクエストを開く

### 貢献してほしい内容

| 領域 | 求めている内容 | ガイドライン |
|---|---|---|
| 既存実装 (`demo/2d/`, `demo/3d/`) | バグ修正、`SPEC.md` との乖離の修正 | 各テーマの `SPEC.md` が唯一の正です。数値の変更はまず Issue で議論してください。 |
| 新しいエンジン実装 | 既存テーマへの新エンジンの追加 | そのテーマの `SPEC.md` に完全準拠すること: 共通HUD、`+`/`-` キーによる負荷制御、画像欠落時のフォールバック、決定的な生成(`Math.random` 不使用)、物理は可能な限り自前実装。`index.html`、`game.js`、`README.md` の3ファイル構成とすること。 |
| 新しいテーマの提案 | 新しいベンチマークテーマ | 「比較したい負荷軸を1つだけ分離する」という設計原則に従うこと。まず Issue で議論してください。 |
| `demo/*/_bench/` | 計測ハーネスの改善 | |
| `docs/i18n/` | 翻訳 | 下記の翻訳ガイドを参照してください。 |

### 開発環境のセットアップ

Node.js は不要です。任意のHTTPサーバを起動するだけです。例:

```
cd demo && python -m http.server 8000
```

`file://` で直接開くと CORS の制約により動作しません。
動作確認には Chrome を推奨します。

### 翻訳ガイド

- リポジトリルートにある英語版ファイルが正(source of truth)です。
- 翻訳は `docs/i18n/{lang}/` 以下に、英語版と同一の構造で配置してください。
- 対応言語: `ja` (Japanese)、`zh-CN` (Chinese)、`ko` (Korean)、`es` (Spanish)。
- 技術用語・コードブロック・URL・ファイルパスは英語のままにしてください。
- 各ファイルの末尾に `Languages:` フッターを付け、現在の言語はリンクなしのプレーンテキストで示してください。

## 行動規範

敬意を持ち、建設的かつ包括的な態度で臨んでください。あらゆる経験レベルの貢献者を歓迎します。
