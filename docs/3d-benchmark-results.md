## 指標の定義

| 列名 | 定義 |
|---|---|
| 負荷レベル | ベンチハーネスが合成キー「+」を送信した回数（0/10/25）。各テーマの負荷軸（小惑星数・剛体数・パーティクル数・レイ本数など）を段階的に増加させる |
| 平均FPS | サンプリング8秒間の平均（ハーネスがrequestAnimationFrameを直接計測） |
| 1% Low FPS | フレーム時間のp99から算出（1000/p99）。カクつきの指標 |
| Objects | HUD表示から抽出した画面上オブジェクト数。3D版は全テーマがHUDに共通ラベル「Objects」を出力するため、全行で取得できている |
| 転送量（KB） | performance resource timing の transferSize 合計。ブラウザキャッシュの影響を受けるため参考値 |
| 初期化（ms） | DOMContentLoaded までの時間 |
| フレーム数 | サンプリング8秒間の実測フレーム数（60fps維持なら約481） |

補足:
- **04 GPUパーティクル（50万粒子）と10 大量レイキャストでは、PlayCanvasだけが負荷レベル25でも60fpsを維持**。一方で02 剛体物理ではPlayCanvasのみ12fps台に急落、03 スキニングではBabylon.jsとともに読み込みタイムアウトと、得意・不得意がテーマで明確に分かれた。
- 03 スキンドキャラ大群を完走できたのはthree.jsとA-Frameのみ（いずれも1000体で60fps維持）。
- 05 広域地形カリングのBabylon.jsは平均FPSこそ60近いが1% Lowが29.9に悪化しカクつきが発生。
- 06 動的シャドウのBabylon.js（レベル25）は平均4.4fpsに対し1% Low 56.5・フレーム数320と指標が矛盾しており、サンプリング中の長時間ストールによる計測異常の疑いがある。再測定が望ましい。
- 初期化時間はPlayCanvasのテーマ01（2234ms）、A-Frameのテーマ02（1727ms）、PlayCanvasのテーマ08（1336ms）が1秒超。
