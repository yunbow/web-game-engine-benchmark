## 指標の定義

| 列名 | 定義 |
|---|---|
| 負荷レベル | ベンチハーネスが合成キー「+」を送信した回数（0/10/25）。各テーマの負荷軸（敵数・剛体数・光源数など）を段階的に増加させる |
| 平均FPS | サンプリング8秒間の平均（ハーネスがrequestAnimationFrameを直接計測） |
| 1% Low FPS | フレーム時間のp99から算出（1000/p99）。カクつきの指標 |
| Objects | HUD表示から抽出した画面上オブジェクト数。HUDの書式が「Objects: N」のテーマ（01/02/04の一部エンジン）のみ取得可能で、それ以外は空欄 |
| 転送量（KB） | performance resource timing の transferSize 合計。ブラウザキャッシュの影響を受けるため参考値 |
| 初期化（ms） | DOMContentLoaded までの時間 |
| フレーム数 | サンプリング8秒間の実測フレーム数（60fps維持なら約481） |

補足:
- **08 パーティクル・魔法エフェクト**: Phaser4がレベル10で25.6fps、レベル25で21.2fpsに低下。Babylon.jsとLittleJSは読み込みタイムアウトで測定不能。
- **11 ダイナミックライティング**: PixiJSがレベル10以降16.8fpsに急落、KAPLAYはレベル25で1% Lowが29.9に悪化。
- **13 大量テキスト描画**: 最も差が出たテーマ。PixiJSは無負荷でも19.7fps・レベル25で0.9fps、Phaser4はレベル10以降10.8fps、Babylon.jsはレベル25で18.8fps。three.js/A-Frameもレベル25で1% Lowが29.9に悪化。LittleJSとKAPLAYのみ全レベルで60fpsを維持。
- **05 LittleJS（レベル10: 51.5fps）と10 A-Frame（レベル10: 7.6fps）**は前後のレベルで60fpsのため一時的な外乱（スパイク）の可能性が高い。
- 初期化時間はエンジン特性が出ており、テーマ13のBabylon.js（2514ms）が突出、テーマ01のPhaser4（1018ms）・three.js（1061ms）・A-Frame（1072ms）も1秒超。
